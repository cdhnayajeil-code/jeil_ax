JEIL_AX 표준 편집 그리드 — 사용 가이드 (단일 출처)

이 문서는 jeil_ax의 모든 편집 가능한 화면 그리드가 따라야 하는 표준이다. 컴포넌트: app/lib/grid.js + app/lib/grid.css · 데모: 그리드/index.html(빌드 주입으로 자기완결 — file:// 더블클릭으로 바로 열림) · 샘플: 그리드/샘플양식.csv 관리 에이전트: grid-standardizer(로컬). 표준이 바뀌면 이 문서·데모·샘플을 함께 갱신한다.

0. 핵심 원칙

  1. 편집 가능한 그리드는 직접 <table> + innerHTML로 만들지 않는다. createGrid()를 쓴다. (순수 읽기 전용 대시보드 표는 예외 — 표준화 대상이 아니다.)
  2. grid.js는 UI만 담당한다. 데이터 조회·저장·발급은 app/lib/api.js(portalApi) 또는 Edge Function 호출을 콜백(onSave/onBulkAction/onRowAction)에서 처리한다. DB·인증 로직을 grid.js에 심지 않는다.
  3. CSS는 .grid__ 스코프만. 전역 table/th/td를 재정의하지 않는다(같은 화면의 다른 표 보호).
  4. 외부 라이브러리·CDN 금지. 바닐라 ESM. 내보내기는 CSV(UTF-8 BOM).
  5. 비밀값·운영 데이터 직접 노출 금지(CLAUDE.md §1). 한국어 UI.

1. 빠른 시작

<link rel="stylesheet" href="./lib/grid.css">
<div id="myGrid"></div>
<script type="module">
  import { createGrid } from "./lib/grid.js";
  const grid = createGrid("#myGrid", {
    keyField: "id",
    selectable: true, search: true, columnFilter: true,
    editable: true, editToggle: true, paste: true, exportCsv: true, copy: true,
    rows: await portalApi.getSomething(),
    columns: [
      { key:"code", label:"코드", pasteKey:"code" },
      { key:"email", label:"이메일", type:"email", editable:true, pasteKey:"email",
        validator:(v)=> !v || /^[^@\s]+@[^@\s]+\.[^@\s]+$/.test(v) || "이메일 형식 오류" },
      { key:"name", label:"담당자", editable:true, pasteKey:"name" },
    ],
    bulkActions: [{ id:"save", label:"💾 저장", btnClass:"ok" }],
    onBulkAction: (id, g) => { /* g.getDirty() / g.getSelected() 로 처리 */ },
  });
</script>

주의(ESM): 위 예제처럼 import를 쓰는 실제 화면은 file://로 직접 열면 모듈 로드가 막힌다 — 정적 서버로 서빙(루트에서 python -m http.server 후 http://localhost:8000/...) 또는 Vercel 배포본에서 확인한다. (데모 그리드/index.html은 예외다 — 빌드가 컴포넌트를 인라인 주입해 자기완결이라 file:// 더블클릭으로도 열린다. 컴포넌트 수정 후엔 python 그리드/_build_html.py 재실행으로 데모를 최신화한다.)

2. columns 스펙

키 설명
key (필수) 데이터 필드명
label (필수) 헤더 표시명
editable true/false 또는 (row)=>boolean (행 조건부 편집)
readOnly true면 편집 모드여도 잠금
type text/email/number/date/select/checkbox
options select용 [{value,label}] 또는 ["A","B"]
validator (val,row)=> true | "에러메시지"
formatter (val,row)=> html — 읽기 모드 셀 표시(pill/배지/버튼). 값은 직접 escape할 것
exportValue (val,row)=> CSV에 쓸 값(미지정 시 원값)
filter false면 컬럼 필터 비활성
sortable false면 정렬 비활성
width 예: "120px"
align "right"/"center"
pasteKey 붙여넣기 자동 인식 힌트: email/name/code/none
cssClass 셀 추가 클래스

행 내 커스텀 버튼은 formatter에서 data-row-act="액션명"을 넣으면 클릭 시 onRowAction(액션명, row)로 전달된다.

3. options & 콜백

options: columns, rows, keyField, selectable, search, columnFilter, sortable, editable, editToggle, editToggleLabels, paste, exportCsv, copy, exportName, stickyHeader, keyboardNav, pageSize, emptyText, loadingText, rowClass, bulkActions.

bulkActions: [{ id, label, btnClass }] — 툴바 버튼. 클릭 시 onBulkAction(id, gridInstance).

콜백 시점
onSave(changes) 저장 트리거 시(호출측에서 사용)
onBulkAction(id, grid) 툴바 일괄 버튼 클릭 — grid.getDirty()/getSelected() 활용
onRowAction(act, row, ev) 행 내 data-row-act 버튼 클릭
onCellEdit(key, field, value, row) 셀 편집
onSelectionChange(rows) 체크박스 선택 변경
onPaste(touchedKeys) 엑셀 붙여넣기 반영 후

4. 인스턴스 메서드

getData() · getSelected() · getSelectedKeys() · getDirty()(→[{key,row,dirtyFields}]) · setRows(rows) · updateRow(key,patch) · refresh() · setLoading(bool) · clearSelection() · clearDirty() · setEditMode(bool) · isEditMode() · validateAll() · hasErrors() · exportCsv(name) · copyToClipboard() · destroy().

5. 엑셀 붙여넣기 규약

편집 모드에서 편집 셀에 Ctrl+V 하면 탭/개행을 파싱해 채운다. 자동 인식: - @ 포함 값 → pasteKey:"email" 컬럼 - keyField 값과 일치하는 값 → 그 행에 정확 매칭(행 순서가 달라도 OK) - 나머지 → pasteKey:"name" 컬럼 - 거래처코드 열이 없으면 붙여넣기 시작 행부터 순서대로

샘플: 그리드/샘플양식.csv (거래처코드,담당자이메일,담당자명).

6. 새 편집 화면 추가 레시피

  1. 화면에 <div id="xGrid"> + grid.css 링크, <script type="module">에서 createGrid import.
  2. 데이터는 portalApi/Edge Function으로 받아 rows로 전달(또는 setRows).
  3. 편집/일괄 작업의 실제 저장은 콜백에서 portalApi.* 또는 callFn(...) 호출 후 setRows로 갱신.
  4. 변경 후 이 가이드의 사례 표에 추가하고, 표준이 바뀌면 python 그리드/_build_html.py로 HTML 재생성.

7. 금지 사항

  • ❌ 외부 그리드 라이브러리·CDN 추가
