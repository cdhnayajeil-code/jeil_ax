5심층 인터뷰 가이드 (3주차 · 수요 상위 3~4개 부서)

대상 선정: 대시보드 집계에서 ① 응답 수 많은 부서 ② 절감가능시간 큰 부서 ③ ERP 데이터 수요 부서 순으로 3~4곳. 부서당 30분, 창구+실무자 동석.

도입 (5분)조사 응답 요약을 보여주며 — "이 업무에 주 ○시간을 쓰신다고 응답하셨는데, 구체적으로 어떤 흐름인지 들려주세요."
업무 흐름 구체화 (10분)① 그 업무의 시작~끝 단계는? ② 어느 단계가 가장 오래 걸리나? ③ 그 단계에서 쓰는 데이터/화면/파일은? (ERP 메뉴명, 엑셀 파일 위치까지 특정)
기대 수준 확인 (10분)① AI가 초안까지만 해주면 충분한가, 완성까지 기대하는가? ② 결과 검증은 누가·어떻게? ③ 주 몇 회·몇 명이 쓰게 되나? (효과 점수 산정 근거)
제약 확인 (5분)① 이 데이터에 개인정보·기밀이 포함되나? ② 지금 외부 AI에 이 업무를 시켜본 적 있나, 결과는? ③ 도입 시 우려는?
인터뷰 기록 규칙 부서당 1장 메모로 정리해 responses 폴더 옆 interviews에 보관 — 항목: 업무명 / 주당시간×인원 / 데이터 위치(ERP 모듈·파일 경로) / 기대 수준 / 민감도 / 과제화 가능 여부. 이 메모가 과제평가 시트의 효과·난이도 점수 근거가 된다.

6평가·우선과제 선정 (4주차)

6-1. 점수 기준 (과제평가 시트 05-③ 입력)

축산정 기준
기대효과 (0~100)주당 절감시간 × 수혜 인원 환산 — 20 미만(소) / 20~50(중) / 50 이상(대). 인터뷰로 검증된 수치 우선
구현 난이도 (0~100)문서 초안·요약·규정 챗봇 10~30 / 정기 리포트 자동화 40~50 / 알림·점검 50~60 / ERP 연계 조회·분석 60~80 / 신규 데이터 확보 필요 80+
가점 (+)여러 부서 공통 수요 +10 / 기존 데이터(ERP·엑셀)만으로 구현 +5 / 효과 측정 가능 +5
감점 (−)개인정보·기밀 민감도 높음 −10 / 외부 데이터 신규 확보 −10 / 법적 검토 필요 −10

6-2. 평가회의 운영 (7/7 화 권장 · 60분)

사전(관리자) — 대시보드 집계 + 인터뷰 메모로 후보과제 7~10건을 평가시트에 미리 입력, 가배점까지 작성.
회의(관리자+수요 상위 부서장) — 화면 공유로 점수 합의 조정 → 사분면 자동 배치 확인 → ①사분면 상위 3~5건 확정.
회의 후 — [우선과제 내보내기]로 AI_우선추진과제_선정안.json 생성 → responses 폴더 보관 + 보고서 작성 착수. ④사분면(보류)·부적합 과제는 제외 사유를 부서에 회신.

7경영진 보고서 골격 (7/10 금)

  1. 조사 결과 요약 — 회신율, 총 응답, 주당 절감가능시간 합계 (대시보드 KPI 캡처)
  2. 수요 분석 — 기능별 수요 순위, 데이터 위치 분포, 우려사항 (대시보드 막대 집계)
  3. 우선 추진과제 3~5건 — 과제별 효과·난이도·담당 부서·기대 ROI (평가시트 사분면+선정 결과)
  4. 2단계 PoC 추진계획 — 우선과제 중 1~2건 시범 구축 범위·기간·예산 (03 기획 로드맵 준용)
  5. 의사결정 요청 — ① PoC 착수 승인 ② 챗봇 포털 B안 확정 ③ 월 예산 상한 ④ 구축 방식(자체/외주) (03 기획 9장 4건)

보고 형식: 대시보드·평가시트 화면을 직접 시연(04 포털 데모와 함께)하면 별도 장표 제작을 최소화할 수 있다.
