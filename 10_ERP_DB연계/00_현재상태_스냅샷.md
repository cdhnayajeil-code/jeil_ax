# 00. ERP DB 연계 — 현재상태 스냅샷

> **기준일: 2026-07-07** · 관리: 최동혁 · 전담 에이전트: `erp-db-link-manager`
> ERP(UNIERP) DB와 AI 포털·사내 타부서 간 연계의 **현황 대장(단일 출처)**. 상태가 바뀌면 여기를 먼저 고치고 §7 변경 이력에 한 줄 남긴다.
> 비밀값(접속 IP/ID/비밀번호)은 기록 금지 — 서버 지칭은 별칭 **"ERP운영DB"** 만 사용.

---

## 1. 한 줄 요약

ERP운영DB(UNIERP, 사외 IDC MS SQL Server 2022)와 AI 포털의 **실데이터 연계는 아직 미착수(화면은 샘플 데이터)** — 유니포인트 협의(P1)는 대기 중이나, **중간 DB(D1-a안, Supabase)의 스키마·적재 경로·ETL 스크립트는 내부적으로 먼저 준비 완료**됐다. `erp_ro`/`etl_meta` 스키마와 1차 4종+테이블사전 DDL이 적용되고 ETL 스크립트(`10_ERP_DB연계/etl/`)가 작성됐으며, **`erp_ro.table_dict`(ERP 테이블 사전)에는 2,658건이 실제로 적재 완료**됐다(로컬 메타 기반, 운영 ERP DB 접속 없음). 다만 ERP운영DB를 실제로 접속해 추출하는 **1차 4종(영업·구매·재고·품목) 실적재는 아직 실행되지 않았다**(`ERP_DB_CONN` 값 미입력 — §6). 현재 위치는 **P0(정책 수립) → P1(유니포인트 협의) 진입 대기 + P3(중간DB) 내부 준비 선행**이다.

## 4. 포털(JEIL AX) 측 연계 현황

| 항목 | 현재 |
|---|---|
| ERP 실연계 | **미연계** — `pages/` 부서 화면은 추출 샘플(마스킹) 데이터로 동작 |
| 포털 데이터화 | Vercel + Supabase 가동(협력사 발주 포털·계정발급) — **ERP 데이터는 미반영** |
| 중간 DB — 스키마 | **적용 완료**(2026-07-07) — 기존 Supabase 프로젝트(`03_중간DB_구축실행기획` D1-a안)에 `erp_ro`(1차 4종: 품목·영업월·구매월·재고일 + `table_dict` ERP 테이블 사전) · `etl_meta`(`batch_run`+`v_last_success`) 스키마와 RLS(사내만 SELECT, anon·vendor 차단) 마이그레이션 적용 |
| 중간 DB — 적재 경로 | **적용 완료** — service_role 전용 RPC(`erp_etl_upsert`/`erp_etl_batch`)로 `erp_ro`를 REST에 노출하지 않고 적재 |
| 중간 DB — ETL 스크립트 | `10_ERP_DB연계/etl/`: `load_table_dict.py` **실행 완료**(2026-07-07, `erp_ro.table_dict`에 ERP 테이블 사전 2,658건 적재 — 컬럼 문서화 452개·11개 모듈, ERP 접속 없음). `etl_run.py`(1차 4종 추출·적재 초안, 유니포인트 뷰 협의 전 화이트리스트 SELECT, `--dry-run` 지원)는 **실행 대기**(§6) |
| ERP 매핑(가설) | `M_PUR_REQ`·`M_PUR_ORD`·`M_CONFIG_PROCESS`·`M_PUR_GOODS_MVMT`·`Q_INSPECTION`·`M_IV` 등 |
| 원칙 | 운영 DB **읽기 전용 + 중간 DB 경유**, 프론트 직결 금지(`CLAUDE.md §4`) |

## 6. 결정 대기·차단 요인

| 항목 | 내용 | 결정 주체 |

| 리포팅 계층 티어 | Tier A(동일 인스턴스 내 리포팅 DB, 즉시) vs Tier B(별도 서버) — 마스터플랜 v2.0 §11 | 경영층 |

| 네트워크 | 파일럿 IP 허용 → 운영 VPN 전환 시점 | 인프라 결정과 연동 |
