# 00. ERP DB 연계 — 현재상태 스냅샷

> **기준일: 2026-07-08**(v0.5, 이전 v0.4 · 2026-07-08 / v0.3 · 2026-07-08 / v0.2 · 2026-07-07) · 관리: 최동혁 · 전담 에이전트: `erp-db-link-manager`
> ERP(UNIERP) DB와 AI 포털·사내 타부서 간 연계의 **현황 대장(단일 출처)**. 상태가 바뀌면 여기를 먼저 고치고 §7 변경 이력에 한 줄 남긴다.
> 비밀값(접속 IP/ID/비밀번호)은 기록 금지 — 서버 지칭은 별칭 **"ERP운영DB"** 만 사용.

---

## 1. 한 줄 요약

ERP운영DB(UNIERP, 사외 IDC MS SQL Server 2022)와 AI 포털의 **중간 DB(D1-a안, Supabase) 1차 5종 실적재는 2026-07-07 완료**됐고, **2026-07-08에는 P4(포털·챗봇 연결)가 착수·부분 완료**됐다. `erp_ro`(품목·영업·구매·재고·발주+테이블사전) 위에 사내 전용 노출 뷰 6종(`v_erp_*`, `security_invoker`)을 신설해 REST 직접 노출 없이 안전하게 열었고, RLS 격리 실측 테스트(JWT 시뮬레이션)로 사내만 통과·협력사/anon 차단을 확인했다. 이 위에서 부서 대시보드 1개 화면(구매 매입집계) 파일럿 실연동과 챗봇(`jeil-chat`) ERP Tool 4종 배포(version 7 ACTIVE)까지 이뤄졌다. 다만 유니포인트 협의(P1, 읽기전용 계정·뷰 스펙)는 여전히 대기 중이고, 현재 적재분은 초안 SQL 기반이라 협의 후 재적재 가능성이 있다. 오늘 재확인한 ERP 원천 dry-run 건수(발주 2,702·품목 58,249·매출 10·매입 63·재고 659)는 중간DB 적재분(2,699/58,246/10/63/656)보다 소폭 늘어 있어, 최신 재적재를 관리자가 `!`로 직접 실행하기로 결정했다(명령 준비, 미실행). 같은 날 **사용자-부서-사원 매핑·대사 계층을 신설·실적재까지 완료**했다 — ERP `Z_USR_MAST_REC`(사용자 마스터, `usr_id`=MS 이메일 SSOT)와 `B_ACCT_DEPT`(부서 마스터)를 대사해 이메일 기준 사용자↔부서↔사원 관계를 조회할 수 있는 스키마·뷰·ETL job을 구축한 뒤, 관리자 직접 실행으로 **실제 ERP 데이터 적재까지 완료**했다: `erp_ro.dept_master_s` 351건, `erp_ro.usr_master_s` 99건. 대사 결과 **정상 매핑 89건**, **불일치 10건**(부서불일치 6·상태이상 3·테스트 1)이며, **불일치는 정상 매핑에서 자동 제외**하는 정책이 확정됐다(마이그레이션 `erp_user_dept_exclude_unmatched`). 이전 결정 대기였던 부서명 처리방침은 이로써 해소됐고, 상태이상 3건(활성계정인데 (퇴사)/(휴직) 표기)만 현업 확인 권고로 남았다(전용 관리 문서 `05_사용자부서_매핑대사.md` 참조). 현재 위치는 **P0(정책 수립) 마무리 → P1(유니포인트 협의) 진입 대기 + P3(중간DB) 내부 준비 계속 + P4(포털·챗봇 연결) 착수(부분 완료, 사용자-부서 매핑 실적재·가동 포함)**다.
