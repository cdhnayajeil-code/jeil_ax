# 02. ERP DB 연계 — 진행상태 (지속 갱신)

> **관리: 최동혁** · 전담 에이전트: `erp-db-link-manager`
> 페이즈·체크리스트·작업로그를 지속 갱신하는 대장. 작업을 수행하면 §2 체크리스트와 §3 작업로그를 함께 갱신하고, 미러(`정책관리/ERP_DB연계/`)에 동기화한다.

---

## 1. 현재 위치

**P0(정책 수립) 마무리 — P1(유니포인트 협의) 진입 대기 + P3(중간DB) 내부 준비 선행** (2026-07-07 기준)

```
[P0 정책수립]──▶[P1 유니포인트 협의]──▶[P2 리포팅 계층]──▶[P3 중간DB+ETL]──▶[P4 포털·챗봇 연결]──▶[P5 민감영역]
   ● 현재            ○ 대기               ○              ◐ 스키마·스크립트 준비    ○                    ○
```

차단 요인: ①경영 티어 결정(Tier A vs B — 마스터플랜 v2.0 §11), ②마스터플랜 승인, ③(P3 실적재) 루트 `.env`의 `ERP_DB_CONN` 값 미입력(`SUPABASE_SERVICE_ROLE_KEY`는 2026-07-07 입력 완료).

P1(유니포인트 협의)이 열리기 전이라도 ERP 접속이 필요 없는 내부 준비(중간 DB 스키마·적재 RPC·ETL 스크립트 작성)는 먼저 진행했다(2026-07-07). 같은 날 `erp_ro.table_dict`에 ERP 테이블 사전 2,658건(컬럼 문서화 452개, 11개 모듈)을 적재 완료했다(로컬 메타 기반, 운영 ERP DB 접속 없음). 실제 ERP 데이터 적재는 여전히 유니포인트 협의(뷰·계정) 이후에만 가능하다.

## 4. 이슈·메모

- **대기(관리자 실행 필요)**: `SUPABASE_SERVICE_ROLE_KEY` 입력 및 `load_table_dict.py` 실행은 완료(2026-07-07). 남은 대기: 루트 `.env`에 `ERP_DB_CONN` 값 입력(사내 OneDrive `ERP_DB`의 기존 접속 문자열 재사용, 읽기 전용 계정 권장) → ①`python etl_run.py --job all --dry-run`(추출 SQL·건수 검증) → ②관리자 확인 후 실적재. 추출 SQL은 유니포인트 뷰 스펙 협의 전 초안이므로 협의 결과에 따라 수정 필요.
- 원천 자산 확인: 사내 OneDrive `최동혁/ERP_DB`의 erp_chat(ERP MSSQL 직결 챗봇, `table-index.json` 2,658테이블·`module-*.json` 452테이블 컬럼 문서·표준쿼리·query-validator)·db변경요청/02_표준쿼리 등은 중간DB ETL 및 추후 챗봇 2단계 Tool의 지식 원천으로 활용 예정(2026-07-07).
