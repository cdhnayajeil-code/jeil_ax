부서 운영 페이지 확장 구축 기획(안)

실제 ERP DB(UNIERP) 기반 부서별 대시보드 4종 — 목업 구축 결과 및 운영 방안

작성일2026. 06. 12.
관리자최동혁 (사업관리팀)
데이터 원천UNIERP DB (읽기전용)
관련문서01·03 기획, AX 통합(06)

EXECUTIVE SUMMARY

  • 챗봇 포털(04)의 부서 운영 페이지를 기존 사업관리팀 2종에서 전 부서 체계로 확장한다. 1차로 인사·구매·영업·자금 4개 부서 대시보드를 실제 ERP DB 구조·데이터 기반 목업으로 구축 완료했다.
  • 모든 페이지는 01 기획의 원칙(ERP 읽기전용·표준 인터페이스·데이터 최소화)을 준수하며, 실제 ERP 테이블 구조와 실데이터 일부(퇴직정산 69.5백만 원, 거래처 매입, 수주 프로젝트 등)를 반영해 "도입 후 모습"을 그대로 보여준다.
  • 개인정보(인사)는 마스킹, 자금 정보는 열람 범위 제한(부서+임원) 등 페이지별 민감도 등급을 정의했다.
  • 현재는 정적 추출(목업) — 2단계에서 게이트웨이가 동일 쿼리를 주기 실행하여 자동 갱신하는 구조로 승계한다(화면 변경 없음).

1구축 배경

니즈조사(02·05·06)와 별도로, 부서별 AI 활용의 최종 모습인 "ERP 데이터를 부서 맞춤 대시보드로 자동 제공"을 미리 체감할 수 있는 실물 샘플이 필요하다. 경영진·부서장이 추상적 기획서 대신 자기 부서의 실제 데이터 화면을 보면 니즈조사 응답 품질과 도입 수용도가 올라간다.

데이터 출처 — ERP_DB 분석 자산 활용 관리자가 운영 중인 ERP 분석 저장소(OneDrive…\최동혁\ERP_DB)의 부서별 폴더(인사·구매·영업·사업관리·회계)에 축적된 실제 테이블 구조 문서·검증 쿼리·추출 데이터를 원천으로 사용했다. DB 접근은 전부 READ-ONLY 원칙(해당 저장소 CLAUDE.md 규칙)을 따른다.

5권한·보안 설계 (03 기획 권한 매트릭스 준용)

페이지열람 범위보안 처리
인사 인원·급여인사팀 + 경영진사번·성명 마스킹, 개인별 급여 미표시(합계·평균만), 원본은 ERP 권한자 한정
구매 매입집계구매팀 + 경영기획단가(PO_PRC) 미표시 — 거래처·총액 수준만
영업 수주현황영업팀 + 사업관리·경영진수주금액 공개, 여신·채권 상세는 자금 페이지로 분리
자금 자금일보자금팀 + 임원 한정가장 높은 등급 — 잠금 처리 기본, 권한 요청제
품목 존재/중복 조회설계팀 · 생산팀 · 사업관리팀 (주관) + 구매·자재팀 (공유)품목 수요 발생 부서(설계 BOM·생산 자재소요·사업관리 발주 의뢰)가 주관, 마스터 정비 주체인 구매·자재팀이 공유 열람. 조회 전용(마스터 변경 불가)
재고 입·출고 현황자재물류팀 (주관) + 구매팀 · 사업관리팀 (공유)수불 주관 부서가 운영, 발주(구매)·프로젝트 정산(사업관리)이 공유 열람. 프로젝트별 수불 누적은 원가 집계와 월 1회 대사
공통 원칙 전 페이지 상단에 목업/검증 전 고지 배너 고정. ERP 원장 수정 기능 없음(조회 전용). 페이지 공개범위는 포털 관리자 콘솔의 부서별 권한 설정을 따르며, 타 부서는 잠금 표시 + 권한 요청 절차.

6포털(04) 반영 내역 및 향후 확장

구분내용
금회 반영부서 운영 페이지 섹션에 5개 카드 추가 — 영업팀 기존 잠금 카드는 실페이지로 연결 전환, 자금 카드는 "임원·자금팀 한정" 표기, 품목 중복 조회(구매·자재) 추가
향후 확장 후보생산(생산실적 일보 요약) · 품질(불량 유형 통계) · 물류(수불 현황) · 설계(도면 이력) — 니즈조사 결과로 우선순위 확정 후 동일 패턴 증설
니즈조사 연계본 4종을 설문 안내 시 "예시 화면"으로 제시 → 부서별 희망 대시보드 수요(Q5)를 구체적으로 끌어내는 마중물로 활용
결론 부서 운영 페이지 4종 구축으로 "ERP 데이터 → 부서 맞춤 화면" 흐름이 실물로 검증되었다. 추가 비용 없이(기존 ERP_DB 분석 자산 재사용) 전 부서 확장이 가능하며, 니즈조사(1단계)와 챗봇 포털(2~3단계)의 수요 견인 도구로 즉시 활용한다.
