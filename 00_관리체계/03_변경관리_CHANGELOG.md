> **관리: 최동혁** · 프로젝트 전체의 의미 있는 변경을 한 줄씩 기록한다.
> 기록 단위: **일자 · 범위 · 대상 · 내용**. 코드 커밋과 별개로, "무엇이 왜 바뀌었나"를 비개발자도 읽을 수 있게 남긴다.
> 세부 코드 변경은 git 이력, 운영현황 변경은 `이관/00_현황스냅샷 §9`, 협력사 요구사항은 `08/00_종합정리_요구사항이력`에 둔다(중복 금지 — 여기는 프로젝트 레벨 요약).

## 1. 문서 수명주기 정책

각 문서는 `00_문서대장`에서 아래 상태로 관리한다.

| 상태 | 의미 | 갱신 빈도 |
|---|---|---|
| **Draft** | 작성 중·검토 전 | 잦음 |
| **Active** | 현행 기준·운영 중 | 변경 시 |
| **Maintained** | 지속 갱신 대장(현황·진행·CHANGELOG) | 수시 |
| **Reference** | 결정 완료·참고용(변경 드묾) | 드묾 |
| **Archived** | 대체됨·보존만 | 없음 |

- 문서를 **이동·이름변경·폐기**하면 `00_문서대장` + 본 CHANGELOG를 함께 갱신한다.
- 폐기 대신 **Archived 표시 + 대체 문서 링크**를 우선한다(이력 보존).
