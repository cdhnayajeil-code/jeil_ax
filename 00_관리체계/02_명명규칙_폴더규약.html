02. 명명규칙 · 폴더규약

기준일: 2026-06-30 · 관리: 최동혁 파일·폴더·문서의 명명과 배치 규칙. 신규 산출물은 이 규칙을 따른다(CLAUDE.md §7 계승·구체화).

1. 파일 명명 규칙

1.1 기본형 — NN_주제_설명.확장자

  • NN = 2자리 정렬 번호(00은 폴더 인덱스/종합문서 예약).
  • 단어 구분은 언더스코어(_), 한국어 우선.
  • 예: 01_AI_관리시스템_도입_기획서.html, 00_니즈조사_홈.html.

1.2 날짜 표기

  • YYYY-MM-DD(문서 본문·이력) 또는 YYYYMMDD(파일명). 예: 2026-06-30, 20260514.

1.3 정당한 예외 (리네임 대상 아님)

패턴 예 사유
진입점 index.html 웹 표준 진입점
도메인 CNAME GitHub Pages 규약
브랜드 도구 JEIL_AX_비용계산기.html 등 제품 브랜딩(현재 09_비용_호스팅/로 묶음)
부서 운영페이지 영업_수주현황_2026.html §2 부서 페이지 규칙
프로젝트 산출물 2025-095-SUL-EC_원가현황_20260514.html 프로젝트코드 우선 식별
통합 산출물 JEIL_AX_포털데모_통합본.html 빌드 산출물(_build_bundle.py)

2. 부서 운영페이지 규칙 (pages/)

  • 형식: [부서]_[기능]_[연도].html (예: 구매_거래처별매입집계_2026.html).
  • 부서명은 01_기준정보_레지스트리 §1 표준 표기를 따른다(인사팀 등).
  • pages/ 파일은 04_챗봇_포털_데모UI.html·통합본·이관/00_현황스냅샷이 경로를 직접 참조한다 → 리네임 시 참조처 동반 갱신 + 통합본 재빌드 필수(함부로 바꾸지 않는다).

3. 배포본 vs 데모본 구분 (의도적 중복)

같은 화면이 두 위치에 존재하는 것은 정당하며 정리 대상이 아니다.

용도 위치 예
기획/데모본 08_협력사발주포털/02_데모/ 협력사_모바일_포털.html
배포본(라이브·포털 연결) pages/ 협력사_모바일_포털.html, 외주발주_검사진행현황_2026.html

변경 시 배포본을 정본으로 삼고, 데모본은 기획 스냅샷으로 유지한다.

4. MD ↔ HTML 쌍 규칙 (단일 출처)

  • 기획·관리 문서는 .md가 단일 출처, .html은 _build_html.py 자동 생성본(손으로 고치지 않음).
  • 적용 폴더: 00_관리체계/, 실제구축준비 자료/이관/, 08/03_실구축기획/.
  • 공통 빌드 로직: 00_관리체계/lib/_html_builder.py. 각 폴더 _build_html.py는 문서 목록·라벨만 정의.
  • index.html(허브)은 자체완결 대시보드 → 빌드 제외, 직접 관리.
