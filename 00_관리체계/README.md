# 관리체계 허브 — JEIL AX AI 포털

> 관리: 최동혁(dh.choi@jeilm.co.kr) · 기준 규칙: 루트 `CLAUDE.md`
> **이 폴더는 프로젝트 전체의 기준정보·문서·변경관리 단일 출처(Single Source of Truth)다.**
> 조직·권한·ERP매핑·프로세스 같은 **기준정보**와 **문서대장·명명규칙·변경이력**을 여기서 관리한다.

## 목적

프로젝트가 정적 데모 → 실구축으로 전환되며 수정·변경이 잦아진다. 흩어지기 쉬운 **기준정보**와 **문서 체계**를 한곳에 모아, 누구든 "현재 기준이 무엇인지" 한 번에 찾고, 변경을 일관되게 기록하도록 한다.

## 이관 관제 센터와의 역할 분리 (중복 방지)

| 폴더 | 담는 것 | 성격 |
|---|---|---|
| **`00_관리체계/`** (본 폴더) | 조직·권한모델·ERP매핑·프로세스단계·명명규칙·문서대장 | **잘 안 바뀌는 기준** |
| `실제구축준비 자료/이관/` | 배포·계정·시크릿 현황·이관 진행상태 | **자주 바뀌는 운영현황** |

서로 링크로 연결한다. 운영 상태(배포/계정)는 `이관/00_현황스냅샷`이 단일 출처다.

## 문서 구성

| 문서 | 역할 | 갱신 |
|---|---|---|
| **index.html** | **총괄 대시보드 + 관리 허브 2탭**(자체완결·직접 관리) | 수시 — AX_DASH 데이터 블록은 **총괄관리자(jeilax-master)** 갱신 |
| **00** 문서대장 | 전체 문서 인덱스·역할·상태·갱신일 | **문서 추가/상태변경 시** |
| **01** 기준정보 레지스트리 | 조직·권한·ERP매핑·협력사·프로세스·환경·시크릿(이름)·계정 | **기준 변경 시** |
| **02** 명명규칙·폴더규약 | 파일/폴더 명명 규칙·예외·폴더 용도·README 템플릿 | 규칙 변경 시 |
| **03** 변경관리 CHANGELOG | 전체 변경이력 + 문서 수명주기 정책 | **변경 발생 시마다** |

## 관리 규칙

1. **기준정보 변경**(조직·권한·ERP매핑·프로세스 등) → `01_기준정보_레지스트리` 갱신.
2. **문서 추가·이동·상태변경** → `00_문서대장` 갱신.
3. **모든 의미 있는 변경** → `03_변경관리_CHANGELOG`에 한 줄(일자·범위·대상·내용).
4. **운영현황 변경**(배포·계정·시크릿) → 여기 말고 `이관/00_현황스냅샷`을 먼저 고친다.
5. **비밀값은 어떤 문서에도 적지 않는다**(`CLAUDE.md §1`). 이름·존재만.
6. **주요 작업 마무리·의사결정 발생 시** → 총괄 서브에이전트 **`jeilax-master`(총괄관리자)** 를 호출해 총괄 대시보드 갱신.

## 총괄 대시보드 · 총괄관리자 운영 (2026-07-16 신설)

`index.html` 기본 탭이 **총괄 대시보드**다: ①현재 위치 콜아웃 ②핵심 지표(KPI) ③영역별 기능 현황(상태·진행률·결정 게이트) ④진행 타임라인(영역 필터) ⑤의사결정 로그(결정됨/대기·예정, D-day) ⑥AI 에이전트 현황.

- **데이터 단일 블록**: index.html 하단 `[AX_DASH 데이터 시작]`~`[AX_DASH 데이터 끝]` 가드 주석 사이의 `AX_DASH` 객체가 유일한 데이터다. **총괄관리자(jeilax-master, 로컬 `.claude/agents/`)만 이 블록을 편집**하고, 블록 밖(CSS·마크업·렌더 JS)은 수정하지 않는다.
- **KPI는 자동 계산**(영역 상태 집계·최근 14일 업데이트·결정 대기 수·최근접 기한 D-day) — 수치를 손으로 적지 않는다. **2주간 타임라인 무기록 시 "최근 14일 업데이트 0"이 정체 신호로 표시되는 것은 의도된 동작**이다.
- **대시보드는 파생 뷰**: 원천은 4곳 — 운영·이관(`이관/00·03`) · ERP 연계(`10_ERP_DB연계/00·02`) · 전체 변경(`03_변경관리_CHANGELOG`) · 의사결정(`11_제품기획/09_ADR`). 불일치 시 원천이 정답이며 대시보드를 원천에 맞춘다.
- **호출 시점**: 주요 작업 마무리 / 진행상황 점검 / 의사결정 등재·전이. 타임라인은 최신 항목을 맨 앞에만 추가(prepend)하고, 120건 초과 시 90일 경과분을 월 요약으로 압축한다(전수 이력은 03_CHANGELOG).

## 빌드
