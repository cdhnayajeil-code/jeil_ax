1도메인 체계 설계

회사 도메인 jeilm.co.kr은 이미 Microsoft 365 테넌트의 주 도메인으로 검증되어 있습니다(메일·계정 사용 중). 포털은 이 도메인 아래 서브도메인을 신설하며, 기존 웹사이트·메일에는 어떤 영향도 없습니다.

주소용도설명
ai.jeilm.co.kr운영전 직원 접속 주소. 그룹웨어처럼 즐겨찾기·사내 공지로 안내
ai-dev.jeilm.co.kr스테이징배포 전 검증·테스트. 검색엔진 차단(robots.txt + noindex), 관리자·파일럿만 접근. 선택이지만 운영 중 수정사고 방지를 위해 권장
확장 예약(지금은 미생성): api.ai.jeilm.co.kr(게이트웨이 분리 시), admin.ai.jeilm.co.kr(관리 콘솔 분리 시) — 초기에는 단일 서버에서 경로(/admin)로 충분
사전 확인 (STEP 0 첫 작업) — jeilm.co.kr DNS는 누가 관리하는가 .co.kr 도메인의 등록 대행사(가비아·후이즈·카페24 등)와 DNS 관리 콘솔 접근 권한자를 확인해야 합니다. 확인 방법: ① 사내 총무/IT 자산 문서, ② whois jeilm.co.kr 조회(등록대행자 표시), ③ 네임서버 확인(nslookup -type=ns jeilm.co.kr) — 네임서버가 ns.gabia.co.kr형이면 가비아 콘솔, Azure DNS형이면 Azure 포털에서 관리. 레코드 추가 권한만 있으면 작업은 10분 내 완료됩니다.

2DNS 레코드 설정 (실작업 값)

호스팅 결정에 따라 두 유형 중 하나로 등록합니다. 둘 다 "레코드 2건 추가"가 전부입니다.

유형 A — Azure App Service 선택 시 (CNAME 방식)

# DNS 관리 콘솔에 추가할 레코드
호스트: ai        타입: CNAME   값: jeil-ax-portal.azurewebsites.net   TTL: 3600
호스트: ai-dev    타입: CNAME   값: jeil-ax-dev.azurewebsites.net      TTL: 3600

# Azure 커스텀 도메인 검증용 (Azure 포털이 안내하는 값 그대로)
호스트: asuid.ai  타입: TXT     값: (Azure가 발급하는 도메인 검증 ID)

유형 B — 사내 서버 / IDC VM 선택 시 (A 레코드 방식)

# 고정 공인 IP가 부여된 경우
호스트: ai        타입: A       값: (서버 공인 IP, 예: 211.xxx.xxx.xxx)   TTL: 3600
호스트: ai-dev    타입: A       값: (스테이징 IP 또는 동일 IP)            TTL: 3600
절차내용
1. 레코드 등록위 값 입력 — 대행사 콘솔 기준 즉시 저장
2. 전파 확인TTL 3600초 기준 보통 10분~1시간 내 전파. nslookup ai.jeilm.co.kr로 확인
3. 서버 바인딩Azure: 포털에서 커스텀 도메인 추가 → 검증 → 바인딩 / 자체 서버: 웹서버(Nginx 등) server_name 설정
4. SSL 적용3장 절차 — 바인딩 후 즉시
5. Entra URI 교체Redirect URI를 https://ai.jeilm.co.kr/auth/callback으로 — 02 문서 2장

3SSL/TLS 인증서

방식비용적용·갱신
Azure 관리형 인증서 (App Service 선택 시)무료포털에서 클릭 한 번으로 발급·바인딩, 자동 갱신 — 관리 부담 0. 권장
Let's Encrypt (자체 서버 시)무료certbot 설치 → 90일 주기 자동 갱신(cron). 갱신 실패 알림만 모니터링
유료 인증서 (대행사 판매)연 5~20만 원불필요 — 위 무료 방식과 보안 수준 동일. 구매하지 않음

공통 정책: HTTPS 강제(HTTP 접속 시 자동 리디렉션), TLS 1.2 이상만 허용, HSTS 헤더 적용. Entra 로그인이 HTTPS를 요구하므로 인증서 없이는 운영 자체가 불가합니다.
