EXECUTIVE SUMMARY

  • 데모(GitHub Pages)로 검증한 AX 포털을 회사 도메인 ai.jeilm.co.kr로 이전하여 그룹웨어처럼 별도 주소로 접속하는 운영 서비스로 전환합니다. 사내·사외 어디서든 웹 접속이 가능하되, Microsoft Entra SSO + MFA가 출입문 역할을 합니다(제로 트러스트 방식 — 그룹웨어와 동일한 접근 모델).
  • 구축 범위는 4대 축 — ① 도메인·서버 인프라, ② MS 연동(로그인은 완료, OneDrive/SharePoint 문서 연계 추가), ③ UNIERP(사외 IDC, MS SQL) 읽기전용 데이터 연계, ④ Claude API 챗봇 게이트웨이.
  • 데이터는 원천(ERP·OneDrive)에 직접 손대지 않고, 포털 측에 읽기전용 사본(중간 DB)과 큐레이션된 문서 라이브러리만 두는 "원천 보호" 원칙으로 관리합니다.
  • 총 일정 약 16주(4개월), 운영비 월 약 50~90만 원(인프라+API, 100명 기준 추산) 수준. 호스팅 위치(Azure 권장)와 ERP 연결 방식 등 의사결정 3건을 먼저 확정해야 착수할 수 있습니다(9장).

5데이터 관리 종합 원칙 — "실제 데이터는 어디에, 어떻게"

포털이 다루는 데이터를 4계층으로 나누고, 계층마다 보관 위치·접근권한·백업·보존을 다르게 적용합니다. 상세 정책표는 04 문서 3장에 있습니다.

계층데이터보관 위치관리 원칙
① 원천UNIERP 운영 DB, OneDrive/SharePoint 원본 문서현 위치 그대로 (사외 IDC, MS 클라우드)포털은 읽기만. 복사·이동·변경 없음. 원천 장애가 포털로, 포털 장애가 원천으로 전파되지 않는 구조
② 연계 사본ERP 요약 중간 DB, AI 지식 라이브러리 인덱스포털 서버 측 DB야간 배치로 갱신되는 읽기전용 사본. 민감 테이블(급여·자금)은 별도 스키마+권한. 유실돼도 재동기화로 복구 가능
③ 생성 데이터대화 로그, 크레딧·사용량, 업로드 파일포털 DB (대화는 보존연한 적용)개인정보 마스킹 후 저장, 보존 1년(권고) 후 파기, 감사 열람은 관리자 2인 원칙, 일일 백업
④ 비밀·설정API 키, Client Secret, DB 계정Key Vault / 서버 환경변수Git·문서·메일에 절대 미포함, 정기 교체(Secret 2026-11월), 접근자 최소화
핵심 — "AI에게 보여줄 데이터만 선별해서 준다" 전사 문서·전체 ERP를 통째로 연결하지 않습니다. OneDrive는 본인 권한 범위 검색(남의 문서는 안 보임) + 회사가 선별한 지식 라이브러리(규정·양식·공지)만, ERP는 부서별 필요 요약 테이블만 단계적으로 개방합니다. 데이터 사고를 구조적으로 차단하면서 니즈조사 결과에 따라 범위를 넓혀가는 방식입니다.
