# 이관 관제 센터 — JEIL AX AI 포털

> 관리: 최동혁(dh.choi@jeilm.co.kr) · 기준 규칙: 루트 `CLAUDE.md`
> **이 폴더는 AI 포털 이관 작업의 단일 출처(Single Source of Truth)다.** 현재 시스템 상태와 이관 진행을 여기에 디테일하게 기록·관리한다.

## 목적

GitHub Pages 정적 데모 → **Vercel+Supabase(무료 데이터화)** → **Azure(운영)** 으로 옮기는 전 과정을 한 곳에서 관제한다. 이후 시스템이 바뀌면 항상 여기를 먼저 갱신한다.

## 문서 구성

| 문서 | 역할 | 갱신 |
|---|---|---|
| **index.html** | 이관 허브 대시보드(가이드 진입점) | 수시 |
| **00** 현재시스템 상태스냅샷 | 배포·자산·데이터·인증·시크릿 현황 대장 | **시스템 변경 시마다** |
| **01** 이관 실행 가이드 (Vercel+Supabase) | 계정 생성 후 단계별 디테일 실행 절차 | 절차 변경 시 |
| **02** Azure 이관 계획 | Supabase→Azure 운영 전환 매핑·절차·롤백 | 운영 전환 시 |
| **03** 이관 진행 상태 | 페이즈·체크리스트·작업로그·이슈 | **작업할 때마다** |

## 관리 규칙

1. 시스템/배포/데이터/계정 변경 → `00_현황스냅샷` 갱신 + 변경이력.
2. 작업 수행 → `03_이관진행상태` 체크리스트·작업로그 기록.
3. 결정/구조 변경 → `01`/`02` 및 상위(`08/03_실구축기획/07`, `CLAUDE.md`) 동기화.
4. **비밀값은 어떤 문서에도 적지 않는다**(`CLAUDE.md §1`). 이름·존재만.
5. ERP 실데이터 미반영(마스킹 샘플만), RLS 전수 ON.

## 빌드

`.md` 수정 후 `python _build_html.py` 실행 → 공유 스타일 HTML 재생성. `index.html`은 자체완결이라 직접 관리(빌드 제외).
