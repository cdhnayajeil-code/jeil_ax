00. 현재 시스템 상태 스냅샷 (현황 대장)

기준일: 2026-06-29 · 관리: 최동혁(dh.choi@jeilm.co.kr) 이 문서는 시스템이 바뀔 때마다 갱신하는 단일 출처(Single Source of Truth) 다. 배포·데이터·인증·계정 상태가 변하면 여기를 먼저 고치고, §9 변경 이력에 한 줄 남긴다. 비밀값(Entra Secret, DB 접속, 키)은 이 문서에 적지 않는다(CLAUDE.md §1). 이름·존재 여부만 기록한다.

1. 한 줄 요약

제일엠앤에스 사내 AI 포털(ERP 연계 챗봇 + 부서 운영페이지 + 협력사 발주·검사 포털). 현재 GitHub Pages 정적 데모로 운영 중이며, 데모에 실제 데이터 쓰기가 필요해져 임시로 Vercel+Supabase(무료)로 데이터화하고 이후 Azure 운영으로 전환하는 이관을 시작했다(전략: 08/03_실구축기획/07).

5. 인증 현황

항목 현재
방식 Microsoft Entra ID (단일 테넌트 jeilm.co.kr), SPA + PKCE(S256)
앱 등록 JEIL-AX-Portal (식별자·Secret은 MS_connect/에만)
토큰 저장 브라우저 localStorage 'jeilax_auth' (offline_access 포함) — 데모 한정, XSS 노출 → 운영서 재설계
MFA 조건부 액세스 "Require MFA for internal users" 자동 적용
redirect URI https://ai.jeilm.co.kr/, …github.io/jeil_ax/ (SPA 플랫폼)
Secret 만료 2026-12-09 (11월 중 갱신 필요 — 미갱신 시 로그인 중단)
협력사 인증 관리자 주도 발급(ERP 거래처 선택→담당자 이메일로 계정 생성·임시비번). Supabase Auth(이메일/비번), 로그인 시 role=vendor+vendor_bp JWT 주입 → RLS 자기거래처만. 자유가입 제거. 계정 안내는 관리자 M365 메일(Graph)

이관 후 인증 분리(데모): 사내=Entra OIDC→Supabase 연결 / 협력사=Supabase Auth(08/03_실구축기획/07 §4).
