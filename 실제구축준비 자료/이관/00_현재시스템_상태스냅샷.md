## 6. ERP(UNIERP) 연계 현황

| 항목 | 현재 |
|---|---|
| ERP | UNIERP, 사외 IDC의 **MS SQL**(별칭 "ERP운영DB") |
| 연계 상태 | **부분 연계 착수(2026-07-08)** — 중간DB(Supabase `erp_ro`) 1차 5종 실적재 완료(발주 2,699·품목 58,246·매출 10·매입 63·재고 656행) + 사내 전용 노출 뷰 6종(`v_erp_*`)으로 부서 대시보드 1개 화면(구매 매입집계)·챗봇(`jeil-chat`) ERP Tool 4종이 실데이터 연동. 나머지 화면은 여전히 추출 샘플(마스킹) 데이터 |
| 민감 데이터 통제(급여) | **실적재 완료·가동(2026-07-08)** — 급여를 민감 모듈 `payroll`로 신설, `dept_erp_scope` 인사팀 전용 + 인사 페이지 서버 게이트(비인사팀 직접 접근 차단), 인사 데모 페이지 개인급여 샘플화+마스킹. 민감 분리 스키마 `erp_secure.hr_payroll_m`(월별 인원·급여총액·퇴직 **집계만**, 개인별·이름·주민번호·계좌 미포함)에 관리자 직접 실행으로 **143행**(2026.01~06) 실적재, 게이트 API **`jeil-hr`(Edge Function) v1**을 배포해 Entra 검증→`dept='인사팀'`/`portal_admin`만→감사 로그→집계 반환. 자금(F 모듈) 실적재는 별개로 경영 승인 대기·미착수, `AI-Portal-HR` Entra 그룹·유니포인트 재직상태 정의는 잔여 |
| 매핑(가설) | `M_PUR_REQ`·`M_PUR_ORD`·`M_CONFIG_PROCESS`·`M_PUR_GOODS_MVMT`·`Q_INSPECTION`·`M_IV` (`CLAUDE.md §4.4`, `08/03/02`) — 세부 컬럼·재고 입출고 분류는 유니포인트 협의 전 가설 |
| 원칙 | 운영 MSSQL **읽기 전용 + 중간 DB 경유**. 상세 진행은 `10_ERP_DB연계/`(단일 출처) 참조 |
