🗄️ ERP 중간 DB 샘플 데이터 실데이터 · 발주담당자 기준

운영 ERP(UNIERP, JEILMNS)에서 실제 추출한 발주·구매(입고/매입) 데이터를 중간 DB(erp_ro) 테이블 형태로 정리한 자료입니다. 원천: ITEM_DOCUMENT(입출고) 추출 _movements.csv + 거래처 마스터 조인. 발주·구매 각 12건 + 발주담당자·거래처·품목·매입유형.

📅 데이터 기준: 2025-10 ~ 2026-01 입출고 (READ-ONLY 추출)
🔒
표기 정책 — 거래처는 부분 마스킹(상호 앞·끝만 표기, 예: 삼성○○티)으로 알아볼 수는 있으나 전체 상호는 가립니다. 발주담당자는 사번·실명 대신 ID(이니셜)로 표기(ERP 등록자 INSRT_USER_ID 기준 · 대표 매핑). 단가·발주/매입 금액은 구간(range)으로 마스킹. 품목·발주번호·전표번호·수량·일자는 실제값. (검색엔진 noindex)
구성 방식 — 설계 문서 07_ERP_챗봇_중간DB_연계설계 기준. 운영 ERP는 야간 배치로만 접근, 대시보드·챗봇은 중간 DB(erp_ro)만 조회. 실제 스키마: 발주 M_PUR_ORD_HDR/DTL · 매입 M_IV_HDR/DTL(M_IV_TYPE) · 입고 M_PUR_GOODS_MVMT(ITEM_DOCUMENT) · 거래처 B_BIZ_PARTNER.
UNIERP MSSQL
① 원천(읽기전용) → 야간배치 → PostgreSQL erp_ro
② 중간 DB(본 화면) → 조회 API/Tool → 대시보드 · 챗봇
③ 소비
원천: 제일엠앤에스 ERP_DB / _movements.csv (ITEM_DOCUMENT READ-ONLY 추출) + 거래처 마스터(B_BIZ_PARTNER) 조인 · 실제 거래 데이터(금액 마스킹).
발주담당자는 ERP 등록자(INSRT_USER_ID) 기준이며 개인정보 보호를 위해 이니셜 ID로 대체(대표 매핑) · 개인정보·민감 컬럼 미연계 · JEIL AX 실구축 준비 · 08 ERP 중간DB 샘플데이터
