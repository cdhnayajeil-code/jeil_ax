1연계 원칙 — 운영 ERP를 지키는 구조

원칙이유
① 운영 DB 직접 조회 금지챗봇·대시보드 조회가 ERP 운영 성능에 영향을 주면 안 됨. 잘못된 쿼리 한 번이 전사 ERP를 느리게 만드는 구조를 원천 차단
② 읽기전용, 필요한 테이블만포털용 DB 계정은 SELECT 전용 + 지정 테이블/뷰만. 쓰기 권한은 어떤 경우에도 부여하지 않음
③ 중간 DB(사본)를 조회 대상으로포털·챗봇은 포털 측 중간 DB만 바라봄 — ERP와 포털이 서로의 장애에서 격리됨. 유실돼도 재동기화로 복구
④ 자유 SQL 생성 금지AI가 임의 SQL을 만들어 실행하는 방식은 금지. 사전 검증된 조회 템플릿(화이트리스트)만 실행 (7장)

3IDC 연결 방식 3안 (호스팅 결정과 연동)

기준1안. IP 허용 + TLS (최소 구성)2안. IPSec VPN (권장)3안. 동일 IDC 배치
방식IDC 방화벽에서 포털 서버 고정 IP만 1433 포트 허용, MSSQL TLS 암호화 강제포털 서버 ↔ IDC 간 사이트 VPN 터널 — DB 포트가 인터넷에 노출되지 않음포털 서버를 ERP와 같은 IDC에 두어 내부망 연결 (01 문서 B안)
보안중 — 포트가 특정 IP에만이라도 인터넷 경유. 계정 유출 시 방어선 약함높음 — 터널 내부로만 DB 통신높음 — 인터넷 비경유
구축 난이도낮음 — 방화벽 규칙 1건중 — 양측 VPN 장비/게이트웨이 설정 (Azure VPN Gateway 월 5~15만 원 추가)중 — 호스팅 이전 수반
판단파일럿·초기 검증용으로 허용◎ 운영 표준 — Azure 호스팅 시 이 방식ERP 연계가 핵심 기능화되면
권고 경로 파일럿(W9~10)은 1안(IP 허용+TLS)으로 빠르게 검증 → 전사 오픈 전(W12)까지 2안(VPN)으로 전환. 어느 안이든 야간 배치 1회 + 소량 조회 트래픽이라 회선 부담은 미미합니다. IDC 측 방화벽 작업은 유니포인트 협조 필수 — 4장 협의 체크리스트에 포함.

4유니포인트(벤더) 협의 체크리스트 — STEP 0에서 조기 발송

No.요청 사항비고
1읽기전용 DB 계정 발급 — SELECT 전용, 합의된 테이블/뷰 한정계정명 예: jeilax_ro. 비밀번호는 Key Vault 보관
2대상 테이블 스키마 자료 — 영업·구매·재고·원가·인사·자금 관련 테이블 정의서(컬럼·코드값)5장 데이터 범위 기준. UNIERP 표준 스키마 문서면 충분
3읽기전용 뷰(View) 생성 협의 — 원테이블 직접 접근 대신 필요한 컬럼만 노출하는 뷰 권장민감 컬럼(주민번호 등) 뷰에서 제외 — 원천에서 차단하는 효과
4IDC 방화벽 정책 — 포털 서버 IP 허용(1안) 또는 VPN 연결(2안) 지원 여부·절차3장 연결 방식 결정과 연동
5야간 배치 조회 허용 시간대·부하 가이드 — 권장 새벽 시간, 동시 세션 제한값운영 영향 0 확약을 위한 합의
6TLS 암호화 연결 지원 버전 확인 (MSSQL Encrypt 옵션)구버전이면 VPN(2안) 필수
7기술지원 비용 견적 — 계정 발급·뷰 생성·방화벽 작업 공수STEP 0 품의에 포함
8(확인) DB 버전·패치 수준, 백업·유지보수 시간대배치 일정이 ERP 백업과 겹치지 않게

6중간 DB·동기화 설계

6-1. 중간 DB 구성 (포털 DB 내 스키마 분리)

-- PostgreSQL 기준 (포털 DB와 동거 시작, 필요 시 분리)
스키마 erp_ro       : 일반 등급 — sales_orders_m(월 수주), purchase_m(매입),
                      inventory_d(재고 일별), project_cost_m(프로젝트 원가)
스키마 erp_secure   : 민감 등급 — cash_daily(자금일보), hr_payroll_m(급여 총액 추이)
                      → DB 권한도 분리: 포털 앱의 일반 역할은 erp_secure 접근 불가
공통 컬럼           : synced_at(동기화 시각), src_updated(원천 기준 시각)

6-2. 동기화 배치 정책

항목정책
주기기본 일 1회 (새벽 03:00). 자금일보 등 당일성 필요 영역만 추후 시간 단위(업무시간 매시) 검토
방식증분 추출 — 원천의 수정일시 컬럼 기준 변경분만. 증분 기준 없는 테이블은 전체 재적재(소량 집계 테이블이라 부담 없음)
변환코드값 → 명칭 변환(부서·거래처 코드), 집계 수준 정리 — "챗봇이 바로 답할 수 있는 형태"로 적재
실패 처리실패 시 관리자 알림(메일/Teams) + 1시간 후 자동 재시도 1회. 포털 화면에는 "데이터 기준 시각" 항상 표시 — 동기화 지연이 사용자에게 보이게
이력배치 실행 로그(시작·종료·건수·오류) 90일 보관 — 감사·장애 분석용
