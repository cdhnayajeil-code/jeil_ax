06. 로드맵 · 단계별 실행 · 운영 전환

작성일: 2026-06-26 · 작성: 최동혁 관련: 00 종합기획, 05 사전정의, 실제구축준비 자료/05_실행체크리스트_로드맵 기준: 전사 AI 포털 16주 마스터 위에 협력사 포털 고유 트랙을 얹는다. (1인 구축 병행 가정)

1. 전체 페이즈 (현재 데모 → 운영 가동)

페이즈 기간(상대) 목표 Go 게이트
P0 사전정의·협의 W1~2 결정·협의·스키마 확정 경영 승인 + 벤더 핵심 회신
P1 기반 구축 W3~6 쓰기 DB·API 골격·인증 경계·파일 로그인+업로드 E2E
P2 ERP 연계 W7~10 발주·검사 배치·대시보드 실데이터·RLS 실데이터 5일 무오류
P3 실시간·알림 W9~11 API 전환·SSE 양방향·알림 다기기 양방향 검증
P4 파일럿 W12~14 협력사 1~2사+사내 2팀 실사용 피드백 반영
P5 오픈·운영 W15~16 전체 확대·운영 캘린더·DR Go-Live 게이트

P2와 P3는 일부 병행. 전사 마스터의 STEP 5(ERP, W9~12)와 자연 정렬되며, 협력사 고유 작업(인증·쓰기·파일·실시간)은 W3부터 독립 트랙으로 선행한다.

P0 (W1~2) — 사전정의·협의

  • 결정 D1~D7 상신, 유니포인트 협의서 발송(핵심 3항 우선).
  • 포털 DB 스키마·API 명세·상태머신 리뷰 확정.
  • 환경·스택·스토리지 골격, Key Vault·gitignore·시크릿 스캔.
  • 게이트: 경영 승인 + 벤더 핵심 회신(매핑·검사 스키마·검수반영).

P1 (W3~6) — 기반 구축

  • sub_portal·sub_meta DB 생성, 백엔드 API 골격(/supplier·/internal).
  • 인증 경계 분리(D1 반영) + 인증 어댑터, BFF/HttpOnly 세션.
  • 파일 업로드 파이프라인(검증·재인코딩·Blob·서명URL).
  • 데모 화면 → API 클라이언트 모듈 1곳으로 연결(목 데이터).
  • 게이트: 협력사 로그인 → 발주 목록 → 사진 업로드 E2E 성공.

P2 (W7~10) — ERP 연계

  • 읽기전용 계정 수령·접속 테스트(TLS), 발주·검사 야간배치(증분).
  • erp_ro 적재 → 대시보드 실데이터 전환(발주구분 그룹·KPI).
  • RLS 2중 강제 적용 + 협력사 격리 테스트(타 거래처 차단).
  • 게이트: 발주·검사 실데이터 연속 5일 무오류 + RLS 차단 확인.

P3 (W9~11) — 실시간·알림

  • localStorage(jeilax_link_v1) → 백엔드 API 완전 전환.
  • SSE 양방향(사진·상태·메시지·검수판정) + 폴링 폴백.
  • 알림 큐·디스패처(Teams/메일/웹푸시), 묶음 정책.
  • 게이트: 서로 다른 기기·계정에서 양방향 실시간 반영 검증.

P4 (W12~14) — 파일럿

  • 협력사 1~2사 + 사내 2팀(구매·품질) 2주 실사용.
  • 주 2회 피드백 수집·반영. 실제 사진·검수 판정 라이브.
  • 게이트: 핵심 피드백 반영 + 중대결함 0.

P5 (W15~16) — 오픈·운영

  • 협력사 확대 온보딩(계정·매핑·교육), 전사 공지.
  • 운영 캘린더 가동, 감사·모니터링·DR 리허설.
  • 게이트: Go-Live 게이트(§3) 전 항목 통과.

3. Go-Live 게이트 (오픈 전 최종 점검)

No. 항목 통과 기준
1 협력사 전체 흐름 로그인→발주조회→사진등록→검수요청→결과수신 무오류
2 사내 전체 흐름 보드 조회→메시지→검수판정→입고진행 무오류
3 행수준 보안 협력사 A가 B 발주 조회/쓰기 차단 확인
4 인증 경계 협력사 토큰으로 /internal 접근 차단
5 파일 보안 위장·초과·악성 파일 거부, 서명URL 만료 동작
6 실시간 다기기 양방향 반영 + SSE 끊김 재동기화
7 비밀값 코드·Git·문서에 시크릿 전무(스캔), Key Vault만
8 데이터 기준 ERP 화면에 동기화 기준 시각 표기
9 감사 조회·판정·업로드 전건 로깅 확인
10 백업·롤백 전일 백업 + 롤백 1회 검증

4. 운영 전환 — 거버넌스·모니터링·DR

운영 캘린더 (반복)

주기 작업
일 야간배치 성공 확인·실패 알림 대응, 업로드/판정 이상 점검
주 오류 로그·응답속도 리뷰, 미처리 검수·메시지 점검
월 사용량·스토리지 리포트, 협력사 계정·매핑 정합성, 감사 로그 점검
분기 RLS·오프보딩 점검, 보존연한 경과 사진 아카이브/삭제, 보안 패치
2026-11 (D1=A안 시) Entra Secret 갱신 → Key Vault 교체
연 DR 복구 리허설, 호스팅·스토리지 비용 재평가

모니터링·알림

  • 다운·배치 실패·스토리지 임계·예산 70% 시 관리자 알림.
  • "데이터 기준 시각" 상시 노출로 동기화 지연 가시화.

백업·DR

  • 포털 DB·사진 스토리지 정기 백업, 중간 DB 장애 시 재배치 복구(ERP와 격리).
  • 스테이징 복원 리허설로 RPO/RTO 검증.

6. 성공 기준 (오픈 후 1분기)

  • 외주 발주의 사진·검수·메시지가 포털 단일 채널로 집중(전화·메신저 분산 감소).
  • 협력사 격리·감사 무사고.
  • 검수 판정 이력·증빙 보존으로 분쟁 추적성 확보.
  • ERP 운영 부하 0 유지(야간 배치만).
