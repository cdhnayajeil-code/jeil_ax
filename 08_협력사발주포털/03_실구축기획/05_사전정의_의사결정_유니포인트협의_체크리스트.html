05. 착수 전 사전 정의 · 의사결정 · 유니포인트 협의 · 체크리스트

작성일: 2026-06-26 · 작성: 최동혁 · 대상 페이즈: P0 관련: 전 문서 + 실제구축준비 자료/03·05 목적: "지금 인지하지 못한 채 착수하면 막히는 것" 을 한곳에 모아, 착수 전에 정의·결정·구성한다.

1. 의사결정 항목 (사람이 정해야 — 코드로 못 푸는 것)

ID 결정 선택지 누가 기한 막히는 것
D1 협력사 인증 방식 A: Entra B2B / B: 자체 인증 (03 비교) 경영진 P0 인증 경계·계정 운영 전반
D2 협력사 계정 ↔ ERP 거래처 매핑 단위 1계정:1거래처 / 1:N / 담당자 다수 구매+관리자 P0 행수준 보안(RLS) 근거
D3 검수 판정의 ERP 반영 범위 포털 기록만 / ERP 참고 반영(사내 수기) 품질+경영 P0 판정 워크플로·법적효력
D4 사진·증빙 보존연한·소유권 보존기간·삭제·분쟁 증거 경영+법무 P0~P1 스토리지 수명주기·약관
D5 파일럿 협력사 선정 외주비중·협조도 기준 1~2사 구매 P0 파일럿 일정
D6 호스팅·파일 스토리지 Azure / 사내 IIS (01 인프라) 경영진 P0 스토리지·VPN·인증서
D7 백엔드 스택 단일 / 하이브리드(권고) 관리자 P0~P1 개발 착수

D1·D3는 본 기획에서 결정 보류(비교·논점만 제시). 나머지는 권고안과 함께 상신.

3. 유니포인트(ERP 벤더) 협의 — 협력사 포털 특화 (조기 발송)

전사 03 문서 협의표에 더해, 협력사 포털에만 필요한 항목을 추가한다.

No. 요청 비고
1 읽기전용 계정(jeilax_ro) — SELECT 전용, 합의 테이블/뷰 한정 비번 Key Vault
2 발주 테이블 스키마 확정: M_PUR_ORD_HDR/DTL(PO_NO·BP_CD·발주구분·납기) 02 매핑 검증
3 발주구분 코드값 정의: SIE/SIE-1/SIF/DIV, SUBCONTRA_FLG 의미 ⑦단계 분기 근거
4 거래처(B_BP) ↔ 협력사 식별 컬럼 — RLS 매핑의 ERP측 키 ★D2 핵심
5 검사 테이블 Q_INSPECTION_*(의뢰·결과번호·합부) + M_PUR_GOODS_MVMT(입고·검사상태) 스키마 02 검증
6 입고/출하/매입 컬럼(MVMT_RCPT·DN_NO·IV_QTY) 프로세스 ⑧~⑩
7 읽기전용 뷰 생성 협의 — 필요 컬럼만, 민감 컬럼 제외 03 보안
8 IDC 방화벽/VPN 정책, TLS 지원 버전 03 전사문서 3장
9 야간배치 허용 시간대·세션 제한 운영영향 0
10 (확인) 검수 판정 결과를 ERP에 반영해야 하는지·방법 ★D3 연동
11 기술지원 견적(계정·뷰·방화벽 공수) 품의

최우선 회신 필요: No.4(거래처-협력사 매핑), No.5(검사 스키마), No.10(검수 반영). 이 3개가 협력사 포털 설계를 막는 외부 의존이다.

5. "지금은 못 보지만 곧 막을" 숨은 항목 (선제 인지)

숨은 이슈 왜 늦게 터지나 선제 대응
거래처-협력사 매핑이 1:N(한 협력사 여러 거래처코드) 데모는 1:1 가정 D2에서 카디널리티 확정, supplier_bp_map N:N 수용
발주 라인(품목)별 사진·검수 요구 데모는 발주(헤더) 단위 화면 요구 재확인, po_line 단위 수용 여부
검수 불합격 후 재작업·재요청 루프 데모는 단순 합/부 상태머신에 재요청 경로(02 §5)
협력사 오프보딩(계약종료) 운영 후 발생 계정·매핑 비활성 절차, 분기 점검(03 §7)
사진 누적 스토리지 비용 시간 경과 후 수명주기·아카이브 정책(D4)
Entra Secret 만료 2026-12-09 고정 기한 D1이 A안이면 갱신 일정 연결(루트 §6)
통합 데모본 수기 수정 빌드 누락 _build_bundle.py로만 재생성(§8.5)
OneDrive 동기화 지연 배포 충돌 배포 시 /tmp clone 빌드(§8.3)
