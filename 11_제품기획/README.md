# 11_제품기획 — 전사 제품기획·아키텍처 단일 출처(SSOT)

> 최종 갱신: 2026-07-08 · 관리자: 최동혁(dh.choi@jeilm.co.kr) · 상태: Draft

## 목적

JEIL AX(사내 AI 포털: ERP 연계 챗봇 + 부서 운영페이지 + 협력사 발주·검사 포털)의 **현재 시점 시스템 구조를 CTO 관점에서 통합·격상한 제품기획 세트**다. 여러 곳에 분산된 아키텍처·데이터·인증·이관 정보를 한 트랙으로 묶어, **개발PM·개발자·CTO가 시스템 맥락과 개발 과정을 한 번에 이해하고 유지·확장·이관할 수 있게** 한다.

이 폴더는 **개념·설계·의사결정의 정본**이다. 운영현황(배포·계정·시크릿)과 세부 실측 기록은 각 영역 SSOT를 **링크로 참조**하며 값을 복제하지 않는다(중복 금지, §관련 문서).

## 단일 출처(SSOT) 규칙

- **이 폴더가 정본인 것**: 전사 통합 아키텍처 개요, PRD/SRS, ERP↔챗봇 연계 설계(P4), Azure 이관 설계, ADR.
- **여기서 참조만 하는 정본(값 복제 금지)**:
  - 운영현황(배포·계정·시크릿·데이터 현황): `실제구축준비 자료/이관/00_현재시스템_상태스냅샷.md`
  - 기준정보(조직·권한·ERP매핑·프로세스): `00_관리체계/01_기준정보_레지스트리.md`
  - 협력사 포털 상세 설계: `08_협력사발주포털/03_실구축기획/01·02·03·08`
  - ERP DB 연계 관제: `10_ERP_DB연계/00~03`
  - 실제 DDL/RLS SQL: `실제구축준비 자료/이관/sql/01~07`

## 갱신 규칙

1. `.md`가 단일 출처. 수정 후 `python _build_html.py`로 `.html` 재생성(HTML은 손대지 않음). `index.html`은 자체완결이라 직접 관리.
2. 설계·의사결정이 바뀌면 `09_ADR`에 항목을 추가하고 관련 문서를 갱신한다.
3. 문서 추가·상태변경 시 `00_관리체계/00_문서대장.md`·`03_변경관리_CHANGELOG.md`를 동시 갱신한다.
4. 비밀값(Entra Tenant/Client ID·Secret, DB 접속문자열·IP, service_role 키)은 어떤 문서에도 적지 않는다.
