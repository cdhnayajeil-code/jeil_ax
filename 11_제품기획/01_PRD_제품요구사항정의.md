## 1. 배경과 문제 정의

- 사내 임직원이 ERP(UNIERP) 데이터를 보려면 ERP 화면·엑셀 추출에 의존해, 접근성이 낮고 부서·권한별 분산이 심하다.
- 협력사와의 발주·검사 과정은 전화·메일·수기 대장으로 진행돼, 진행상태·검사증빙이 실시간으로 공유되지 않는다.
- 데이터 접근을 **안전하게(읽기전용·권한격리) + 대화형(챗봇) + 실시간(협력사 포털)** 으로 통합할 단일 창구가 없다.

## 2. 제품 목표(Goals)

| G# | 목표 | 성공의 모습 |
|---|---|---|
| G1 | ERP 데이터 대화형 접근 | 직원이 챗봇으로 발주·매출·재고 요약을 즉시 조회 |
| G2 | 협력사 발주·검사 디지털화 | 협력사가 모바일로 발주 확인·사진 등록·검수요청, 사내가 실시간 검수판정 |
| G3 | 부서 운영페이지 | 부서별 KPI·현황을 ERP 사본 기반으로 시각화 |
| G4 | 안전한 데이터 경계 | ERP 무영향(읽기전용), RLS 격리, 비밀값 분리 |
| G5 | 확장·이관 가능한 구조 | 스택/호스팅 교체(예: Supabase→Azure)에도 재구축 없이 어댑터 교체로 대응 |

## 3. 페르소나(Persona)

| 페르소나 | 설명 | 핵심 니즈 |
|---|---|---|
| 사내 임직원(일반) | Entra 계정 보유 직원 | 챗봇으로 내 업무 데이터 빠르게 확인 |
| 부서 담당자 | 영업·구매·인사·자금·자재·외주 | 부서 대시보드에서 현황·추이 파악 |
| 사내 관리자 | 협력사·챗봇·계정 운영자 | 협력사 계정발급, 검수판정, 사용량·비용 관리 |
| 협력사(사외) | 발주받는 외주 업체 | 자기 발주만 확인, 사진 증빙 등록, 진행상태 |
| CTO/개발PM | 시스템 총괄 | 구조 이해, 리스크·의사결정 추적, 이관 관리 |

## 4. 범위(Scope) — 기능 영역

### 4.1 In-Scope (제품이 다루는 것)

1. **챗봇(jeil-chat)**: Entra 로그인 사용자 대상 업무지원. 사전 등록 읽기전용 Tool로 포털/ERP 데이터 요약 응답. 사용량·비용 감사.
2. **협력사 발주·검사 포털**: 협력사 모바일(발주 조회·사진 등록·검수요청·메시지) ↔ 사내 대시보드(상태 보드·사진 검수·합/부 판정·메시지). 실시간 양방향.
3. **부서 운영페이지**: 영업 수주·구매 매입·인사 인원급여·자금 일보·자재 입출고·품목 조회·프로젝트 원가.
4. **계정·권한 운영**: 협력사 계정 발급/승인/비번초기화, 사내 RBAC(Entra 보안그룹 기반, 계획).
5. **ERP 데이터 연계**: ERP→중간DB 야간배치, 중간DB→앱/챗봇 노출(설계 → [06](06_ERP연계_챗봇활용_설계.md)).

### 4.2 Out-of-Scope (이번 제품이 하지 않는 것)

- ERP 운영DB에 **쓰기/직접조회**(절대 금지, [CLAUDE.md §1·§4]).
- 금전·발주 확정·전자결재 등 **실거래 액션 자동 실행**(조회·집계·초안까지만).
- 자체 회원가입·자체 비밀번호 정책(사내는 Entra SSO 단일 경로).
- 문서 RAG(3단계) — 로드맵에만, 이번 범위 밖.

## 7. 성공 지표(Success Metrics)

- **채택**: 챗봇 주간 활성 사용자 수(Entra UPN 기준, `chat_log`), 협력사 포털 로그인/등록 건수.
- **효율**: 발주·검사 1건당 처리 리드타임(수기 대비 단축), 사진 검수 소요.
- **안전**: 보안 advisor 심각(ERROR/WARN) 0 유지, ERP 운영 부하 이벤트 0.
- **비용**: 챗봇 월 토큰·USD 추정비용 예산 내.

## 8. 제약·가정(Constraints & Assumptions)

- ERP 스키마 접근·뷰 생성은 **벤더 유니포인트 협의 선행**(임의 진행 금지).
- 현재 ETL 추출 SQL·ERP 매핑은 **유니포인트 확정 전 "가설"** 상태([09 ADR](09_ADR_의사결정기록.md)).
- 협력사 인증 방식(D1), 백엔드 스택, 호스팅(Azure vs IIS), 토큰 보관(BFF)은 **미확정** — 09 ADR에 기록.
- 배포는 OneDrive 직접 push 금지, `/tmp` clone 절차([CLAUDE.md §8]).
