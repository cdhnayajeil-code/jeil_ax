00 · 제품기획 개요 — JEIL AX 사내 AI 포털

관리자: 최동혁(dh.choi@jeilm.co.kr) · 기준일: 2026-07-08 · 상태: Draft 관련: 01 PRD · 03 아키텍처 · 08 보안 · 09 ADR

이 문서는 제품기획 세트의 관문(entry point) 이다. 처음 합류한 개발PM·개발자·CTO가 이 문서 하나만 읽어도 JEIL AX가 무엇이고, 지금 어디까지 와 있으며, 나머지 문서에서 무엇을 찾을 수 있는지 파악하도록 구성했다.

1. 한 줄 정의

제일엠앤에스(JEIL M&S) 사내 AI 포털 — Microsoft Entra(Teams/MS365) 계정으로 로그인하여, ① ERP(UNIERP) 데이터를 활용하는 업무지원 챗봇(jeil-chat), ② 부서별 운영 관리페이지, ③ 협력사 발주·검사 포털을 한곳에서 제공한다. 운영 도메인 ai.jeilm.co.kr.

2. 제품 비전과 현재 위치

  • 비전: "사내 임직원과 협력사가 ERP 데이터에 안전하게, 필요한 만큼, 대화형으로 접근하는 단일 창구."
  • 현재 위치(2026-07-08): 정적 HTML 데모 → 실제 구축 전환 중. 협력사 발주·검사 포털과 챗봇 게이트웨이는 Supabase 기반으로 실가동, 부서 운영페이지는 대부분 정적 샘플, ERP→중간DB 연계는 파일럿 실적재까지만 완료(앱/챗봇 미연결).
  • 상세 운영현황(배포·계정·데이터)은 값 복제 없이 → 이관/00_현재시스템_상태스냅샷 참조.

4. 3계층 데이터 영역 (전체를 관통하는 핵심 구조)

JEIL AX의 모든 설계는 세 데이터 영역의 분리에서 출발한다. 이 경계를 이해하면 나머지가 따라온다.

①  ERP 운영DB (원천)              ②  중간 DB (읽기 사본)                 ③  포털 DB (자체 생성)
┌─────────────────────┐       ┌──────────────────────────┐       ┌──────────────────────────┐
│ UNIERP · MSSQL 2022  │       │ Supabase PostgreSQL       │       │ 동일 Supabase 프로젝트     │
│ 사외 IDC · 단일노드   │ ETL   │ 스키마 erp_ro / etl_meta   │       │ 스키마 public              │
│ DB명 JEILMNS         │──읽기─▶│ (ERP 집계 사본)            │       │ sp_*(발주포털)·vendor_*    │
│ 2,658 테이블         │ 야간   │ 파일럿 실적재 완료          │  ✕??  │ ·chat_log                  │
│ 벤더 유니포인트 관리  │ 배치   │ 앱/챗봇엔 아직 미연결(P4)   │       │ 실가동 중                  │
└─────────────────────┘       └──────────────────────────┘       └──────────────────────────┘
   쓰기·직접조회 금지               읽기전용 · RLS · REST 비노출            RLS 격리 · Realtime · Storage
  • ① → ② 는 ETL 배치(읽기전용 SELECT + NOLOCK + 파라미터 바인딩, 화이트리스트 SQL 5종). 프론트/챗봇은 ①에 절대 직접 접근하지 않는다.
  • ② 와 ③ 은 같은 Supabase 프로젝트에 있으나 현재 단절되어 있다. 앱과 챗봇이 실제 조회하는 것은 ③(public.sp_*)뿐이고, ②(erp_ro)는 아직 화면·챗봇에 노출되지 않았다. 이 P4 단절 구간을 잇는 것이 다음 핵심 과제다 → 06 ERP연계.
  • 자금·인사 등 민감영역(erp_secure)은 P5로 예약, 미생성.

5. 세 개의 인증 경계 (요약)

사용자 인증 토큰 저장 격리
사내 직원 Microsoft Entra ID (OAuth PKCE 직접구현) localStorage jeilax_auth Entra 도메인 @jeilm.co.kr
협력사(사외) Supabase Auth (이메일/비번, 관리자 발급) localStorage jeilax_sb_auth RLS로 자기 거래처(bp_cd)만
사내 관리자 Supabase 세션 + portal_admin 등록 (동일) role=internal + 관리자 테이블

사내(Entra)와 협력사(Supabase Auth) 경계는 의도적으로 분리된다. Edge Function이 Entra 토큰을 MS Graph로 검증하는 브리지 역할을 한다. 상세 → 03 아키텍처 §인증.

6. 기술스택 현황 (2026-07-08 사실)

계층 현재 실제 비고
프론트 바닐라 JS + HTML(ESM 모듈, 번들러 없음) app/, 동적 pages/ 2종. Supabase JS는 CDN(esm.sh)
백엔드 Supabase Edge Functions(Deno/TypeScript) 5종 계정발급·승인·비번초기화·챗봇·챗봇관리
DB Supabase PostgreSQL public + erp_ro/etl_meta
인증 Entra ID(사내) + Supabase Auth(협력사)
챗봇 LLM OpenAI gpt-4o-mini(현행) UI 표기는 "Claude" — 불일치, 09 ADR 기록
ETL Python(pyodbc + ODBC Driver 17) 사내 PC 실행, RPC로 적재
배포 Vercel(jeil-ax, ai·ax 둘 다 main 자동배포) Azure 이관 계획만 수립

스택은 아직 "확정"이 아니다. CLAUDE.md §3는 후보 A(.NET/MSSQL)·후보 B(Node/Postgres)를 비교하며 하이브리드를 권고하고, 현재 실동작은 Supabase다. 최종 확정은 09 ADR 미결정 항목.

8. 용어집(Glossary)

용어 뜻
ERP운영DB UNIERP의 운영 MS SQL(사외 IDC, DB명 JEILMNS). 문서엔 별칭만 표기
