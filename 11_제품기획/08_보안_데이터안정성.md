## 1. 보안 경계 4원칙 (현행 준수 사실)

| 원칙 | 구현 | 검증 |
|---|---|---|

| 위협 | 표면 | 대응 |

### 3.2 보안 Advisor(`get_advisors security`)

| 수준 | 항목 | 대상 | 판정·조치 |
|---|---|---|---|
| WARN | SECURITY DEFINER 함수를 anon/authenticated가 REST로 실행 가능 | `public.update_my_vendor_contact` | **점검 필요**: 세션 없는 호출 무효화·자기 행만 수정하는 내부 가드 재확인. 불필요 시 EXECUTE 회수 또는 SECURITY INVOKER 전환 검토 |

### 3.3 성능 Advisor(`get_advisors performance`, 총 136건)

| 항목 | 건수 | 성격 |
|---|---|---|
| `multiple_permissive_policies` | 125 | 여러 permissive 정책 중복(사내 `internal_all`+`vendor_own`) — 쿼리마다 복수 정책 평가. **보안결함 아님**, 최적화 여지 |
| `auth_rls_initplan` | 8 | RLS가 행마다 `auth.<fn>()`/`current_setting()` 재평가. `(select auth.uid())` 래핑으로 최적화 권장 |
| `unused_index` | 2 | 미사용 인덱스(예: `idx_sp_message_bp_created`) |
| `unindexed_foreign_keys` | 1 | `vendor_application_matched_bp_cd_fkey` 커버링 인덱스 없음 |

> 성능 항목은 **취약점이 아니라 튜닝 대상**이다. 데이터가 커지기 전(현재 소량) 정리하면 이관·확장 시 유리. RLS initplan 최적화는 [09 ADR](09_ADR_의사결정기록.md) 개선 항목.

## 6. 개인정보 처리

- 급여·주민번호·인사평가 등은 **권한 그룹 밖 화면/로그/캐시에 미노출**.

- 챗봇 시스템 프롬프트가 개인정보·비밀값 응답을 금지.

## 7. 데이터 안정성·백업/DR

- **기준시각**: ERP 파생 데이터는 `etl_meta.v_last_success` 표기 → 배치 실패 시 마지막 성공분 유지·인지.
- **백업**: Supabase 자동 백업(플랜 의존). 운영 전환 시 중간DB·포털DB 백업·복구 정책 수립 필요([CLAUDE.md §11-E]).
- **무결성**: ETL은 추출=적재(유실 0) 확인 방식. 배치 이력으로 재적재 추적.

## 9. 전체 점검 (2026-07-23, 코드리뷰 + 라이브 실측)

> 범위: Supabase 보안·성능 어드바이저 전량(보안 24건·성능 147건 **모두 확인**), RLS/SECURITY DEFINER 실측, 요청접수 신규 코드 리뷰, 저장소↔배포 정합성, 시크릿 노출.

### 9-1. 오탐 정리 — 건전함이 확인된 것

경보 대부분은 **설계 의도**였다. 실제로 검증한 근거를 남긴다.

| 어드바이저 경보 | 실측 결과 | 판정 |
|---|---|---|
| `rls_enabled_no_policy` 15건 (INFO) | `chat_*`·`portal_request`·`erp_load_scope`·`perm_*`·`ai_*` — RLS 켜고 정책 0 + anon/authenticated 권한 회수 | ✅ 의도된 **fail-closed**(Edge Function 전용) |
| `multiple_permissive_policies` **125건** (WARN) | 정책이 `{public}` 롤 대상이라 anon까지 열거될 뿐, 실제 게이트는 `is_internal()`·`vendor_bp()` | ✅ 성능 노이즈. 보안 구멍 아님 |
| SECURITY DEFINER 공개 실행 4건 (WARN) | `vendor_master_sync`=portal_admin 검사 후 예외, `update_my_vendor_contact`=`auth.uid()` 없으면 0행, `erp_finance_overview`=`perm_effective` finance 검사, `perm_can`=판정 함수 | ✅ 전부 **내부 인증 가드 보유** |
| 협력사 경계 | `is_internal()`·`vendor_bp()`가 **`app_metadata`**(서버 통제, 사용자 조작 불가) 기반 | ✅ 건전 |
| 시크릿 노출 | 추적 파일 전체 스캔 — JWT/`sk-`/service_role 패턴 **0건**, `.gitignore` 정상 | ✅ |

### 9-2. 발견 — 조치 필요

| # | 심각도 | 발견 | 근거(실측) |
|---|---|---|---|
| F1 | **높음** | **저장소 ↔ 배포 드리프트** — `jeil-chat` 저장소는 P2b~P2d 포함(`791d949`), 운영은 **v28**(미포함). 코드만 보고 "동작한다"고 오판할 위험 | `git log` vs `list_edge_functions` |
| F2 | **높음** | **관리자 1명(SPOF)** — `portal_admin` 1행. 완료 게이트 도입으로 관리자 의존이 더 커졌는데 백업이 없다. 이 계정이 막히면 요청 처리·권한 부여가 전면 정지 | `portal_admin`=1 |
| F3 | **높음** | **`erp_load_scope` 상태 변경에 감사 흔적 없음** — `updated_by` 컬럼 부재. 이 값이 **데이터 요청 완료 게이트를 여는 통제**인데 누가 언제 바꿨는지 남지 않는다(`portal_request`는 `assignee_upn`을 남김) | DDL 확인 |
| F4 | **중간** | **강제 완료(`force`) 흔적 미보존** — `fulfilled=false`인데 강제 종결해도 `handled_note` 자유텍스트 외에 구분 수단이 없다. 사후에 "정상 완료"와 "미반영 강제 종결"을 못 가른다 | 코드 리뷰 |
| F5 | **중간** | **개인정보 보존기간 미이행** — `chat_retention_days=180` 설정만 있고 **실제 파기 배치 없음**(`pg_cron` 미설치). 최고령 로그 2026-07-07이라 현재는 위반 아니지만 **2027-01 초부터 정책과 실제가 어긋난다** | `pg_extension`=0, `chat_log` 최고령 2026-07-07 |
| F6 | **중간** | **`erp_ro` 모듈 차등이 RLS에 없음** — 정책은 "`role=internal`이면 전량 SELECT". 현재 **internal JWT 보유자 0명**이라 실제 노출은 없으나, 사내 Supabase 세션을 도입하는 순간 앱 게이트를 우회해 매출·매입·발주 전량 열람이 가능해진다 | `auth.users` app_role: vendor 4 / none 1 |
| F7 | **중간** | **`jeil-chat` 116KB 단일 파일** — 배포 경로를 제약하고(§17-1) 변경·리뷰 위험을 키운다 | 파일 크기 |
| F8 | **낮음** | **상수 중복** — `KIND_KO`·상태 라벨이 `jeil-chat`·`jeil-portal-request` 양쪽에 복제. 유형 추가 시 한쪽만 고칠 위험 | 양쪽 정의 확인 |
| F9 | **낮음** | `portal_request_touch` **search_path mutable** | 보안 어드바이저 |
| F10 | **낮음** | `vendor_master_sync`·`update_my_vendor_contact` **anon EXECUTE 부여** — 내부 가드로 무력화되나 방어심도상 REVOKE 권장 | ACL 확인 |
| F11 | **낮음** | **유출 비밀번호 차단 비활성** — 협력사 계정 4개가 자체 비밀번호 사용 | 보안 어드바이저 |
| F12 | **정보** | 동조자에게 **원 요청자의 사유가 노출** — `mine`/`get_my_requests`가 supporters 참여 건의 `reason`을 그대로 반환. 공동 요청 성격상 의도로 볼 수 있으나 문서화 필요 | 코드 리뷰 |
| F13 | **정보** | `auth_rls_initplan` 16건 — `auth.jwt()`를 `(select auth.jwt())`로 감싸면 해소(행 단위 재평가 제거) | 성능 어드바이저 |
