동작 방식 — ERP(UNIERP) 운영DB → 야간/수동 배치(ETL, 읽기전용) → Supabase erp_ro 스키마 적재 → 사내 전용 뷰 public.v_erp_*로 화면·챗봇에 제공. erp_ro는 REST에 직접 노출되지 않으며(보안), 이 페이지는 public.v_erp_sync_overview 뷰를 조회합니다. 데이터는 유니포인트 매핑 확정 전 파일럿이며, 최신 연동 시점 이후의 ERP 변경은 다음 배치에서 반영됩니다.

연결 정보

원천 시스템UNIERP · MS SQL Server(사외 IDC) — DB JEILMNS
접속 방식ODBC(SQL Server Driver 18/17) · 읽기전용 조회만(SELECT … WITH (NOLOCK)) · 화이트리스트 SQL 고정, DDL/DML 없음
적재 경로ETL 스크립트(10_ERP_DB연계/etl/etl_run.py) → Supabase RPC erp_etl_upsert(민감분 erp_secure_upsert) → erp_ro/erp_secure
실행 방식수동 실행(자동 스케줄러 미도입) — 배치 이력은 etl_meta.batch_run에 기록. 상단 표의 "최신 연동"이 각 소스의 마지막 성공 시각
접속 정보 보관서버·계정·비밀번호는 소스·문서에 기재하지 않음 — 실행 환경의 .env(git 제외)에만 보관
데이터 격리RLS로 사내(internal) 세션만 조회 · 협력사·anon 0행 · 급여 등 민감분은 erp_secure 분리 후 전용 RPC로만 접근(접근 감사 기록)
