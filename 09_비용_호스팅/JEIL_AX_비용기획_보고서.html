요약 (Executive Summary)

전사 AI 사용 방식은 실제 쓴 토큰만큼 내는 API 방식과 사람 수 × 정액인 좌석제로 나뉜다. 저사용 인원이 많은 전사 환경에서는 API 방식이 총비용이 낮고, 부서·개인별 사용량 추적과 별도 과금이 가능하다. JEIL은 이미 사내 포털(ai.jeilm.co.kr)을 API로 구축 중이므로 API 게이트웨이 중심 + 헤비유저 분리 과금 구조가 가장 적합하다.

핵심 결론 — ① 주력은 API 방식, 헤비/전문 사용자만 선택적 좌석·쿼터를 주는 하이브리드가 최적. ② 헤비유저 별도 과금은 기술적으로 가능(Admin Usage API·Workspace 한도). ③ 모델 라우팅으로 단일 모델 대비 25~40% 추가 절감.

＋입력/출력 토큰 비용 분해 & 사용자별 산정 기준

산정 기준 (계산식)

  • 월 토큰 = 일 질의 수 × 22 근무일 × (질의당 입력 토큰 + 출력 토큰)
  • 유효 입력 토큰 = 입력 × (1 − 캐싱률) + 입력 × 캐싱률 × 0.1  (캐시 적중분 90% 할인)
  • 월 비용 = (유효 입력/1M × 입력단가) + (출력/1M × 출력단가)
  • 적용 단가: Sonnet 4.6 $3 / $15 · 캐싱률 40% · 질의당 입력 3,500 / 출력 1,000 토큰 가정

JEIL 적용 설계 (사내 포털 기준)

  1. 모든 요청에 사번·부서 태그를 붙여 사용량 DB(ERP MSSQL 또는 별도 테이블)에 기록
  2. 월 기본 쿼터(예: 1인 200만 토큰/월) 부여 후 초과분 정책 적용
    • 옵션 A — 부서 차지백: 초과 토큰 비용을 부서 비용센터로 배분
    • 옵션 B — 승인제 상향: 초과 시 관리자 승인 후 한도 증액
    • 옵션 C — 좌석 전환: 상시 헤비 사용자는 Team Premium 좌석으로 정액화
  3. 부서·개인별 Top 사용자·월 비용 추이 대시보드 시각화 (니즈조사 대시보드 인프라 재활용)
분리 과금 구축 비용
  • 공급사 측 추가 요금: 없음 (Usage API·한도는 기본 기능)
  • 사내 구축(1회성): 로깅·쿼터·대시보드 개발 공수, 기존 포털 기능 추가 수준이면 수일~2주, 별도 라이선스 비용 없음
  • 운영: 사용량 DB 비용 미미 · 헤비유저 Premium 좌석 전환 시 좌석당 연납 $100/월(약 ₩14만) 추가

7권고 기획안

하이브리드 + API 게이트웨이 중심 모델

  1. 기본은 API 방식 — 사내 포털이 Claude/GPT 중계, 전 직원 접근, 저사용자 무비용
  2. 모델 라우팅 — 단순 Haiku/GPT-5.5 mini → 일반 Sonnet → 고난도 Opus/GPT-5.5, 캐싱·출력 가이드 병행
  3. 사용량 거버넌스 — 사번·부서 태깅 로깅 + 월 기본 쿼터 + 초과 시 차지백/승인제
  4. 헤비유저 선별 좌석화 — 매일 대량 사용하는 전문 인력 소수만 Team Premium 정액화
  5. 월 예산 가드레일 — Workspace/Project 지출 한도로 전사 월 상한 하드캡 설정
예상 200명 총비용: 월 약 ₩296만(API 라우팅) + 헤비 좌석 일부 ≈ 월 ₩300만~400만, 연 ₩3,600만~4,800만 수준에서 운영 가능.

8가정 및 주의사항

  • 단가는 2026-06 공개 기준이며 공급사 정책에 따라 변동 가능. 계약 전 콘솔/영업 최종 확인 필요
  • 환율 ₩1,400/USD 가정. 실제 청구는 USD 기준이라 환율 변동 시 KRW 비용 변동
  • 사용 프로파일(일 질의 수·토큰량)은 추정치 → 1~2개월 파일럿 로그로 보정 권장
  • Claude Enterprise는 비공개 협상가(시트당 약 $60, 일부 최소 70석)로 대규모 전환 시 별도 견적 필요
  • 좌석제는 일정 사용량 포함되나 한도 초과분은 API 단가로 추가 과금될 수 있음(플랜별 상이)
