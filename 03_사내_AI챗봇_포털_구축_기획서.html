Claude API 기반 — 직원별 할당량 관리형 사내 챗봇 서비스

작성일2026. 06. 11.
관리자최동혁
적용규모30~100명
관련문서AI 관리시스템 도입 기획(01)

EXECUTIVE SUMMARY

  • 직원들이 개별적으로 외부 AI를 쓰는 대신, 회사가 통제하는 단일 챗봇 포털을 구축하여 Claude API를 통해 AI를 제공하고, 직원별 사용 할당량(크레딧)을 부여·관리합니다.
  • API 키는 직원에게 배포하지 않고 중앙 게이트웨이 서버에만 보관합니다. 직원은 사내 계정으로 로그인해 웹 챗봇만 사용하며, 게이트웨이가 할당량 차감·로깅·정보 필터링을 수행합니다.
  • 예상 비용은 사용량 기반으로 100명 전사 적용 시 월 약 $210~400 수준(약 30~56만 원)으로 추산되며, Workspace 월 지출한도 설정으로 예산 초과를 원천 차단할 수 있습니다.
  • 1단계 니즈조사 결과를 반영한 부서별 프롬프트 템플릿을 탑재하고, 2단계에서 UNIERP 읽기전용 데이터 연계를 추가합니다.

1추진 배경 및 용어 정리

01 기획서에서 정의한 "데이터 연계형 AI 관리시스템"의 사용자 접점(front-end)으로서, 전 직원이 사용하는 챗봇 형태의 웹 포털을 구축합니다. 포털은 일반 업무지원(문서 초안, 요약, 질의응답)부터 시작해, 단계적으로 ERP 연계 조회 기능을 흡수합니다.

용어 정리 — Claude Code vs Claude API Claude Code는 개발자가 터미널에서 사용하는 코딩 도구(CLI)이고, 챗봇 페이지 구축에 사용하는 것은 Claude API(Messages API)입니다. 본 기획의 과금·할당량 관리는 Claude API 기준이며, Claude Code는 추후 개발팀 생산성 도구로 별도 검토 대상입니다.

설계 3원칙

① 키는 서버에만, 직원에겐 화면만

API 키를 개인에게 배포하면 유출·통제불능 위험. 게이트웨이가 모든 호출을 대행하고 직원은 로그인 기반 웹 화면만 사용합니다.

② 이중 한도 (개인 크레딧 + 전사 지출한도)

개인별 월 크레딧으로 사용을 배분하고, Anthropic 콘솔 Workspace 월 지출한도로 전사 예산 초과를 원천 차단합니다.

③ 모델 자동 라우팅

일반 질의는 저비용 모델(Haiku), 분석·긴 문서는 상위 모델(Sonnet)로 자동 배정하여 품질과 비용을 동시에 관리합니다.

4직원별 할당량(크레딧) 설계

토큰 단위는 직원이 이해하기 어려우므로, "크레딧"으로 추상화합니다. 1크레딧 = 평균 질의 1회(입력 약 2,000 / 출력 약 800토큰 기준). 게이트웨이가 실제 토큰 사용량을 크레딧으로 환산·차감하고, 화면에 잔여량을 표시합니다.

등급대상월 크레딧사용 가능 모델1인 월 예상비용*
라이트일반 직원 (기본 부여)300회Haiku 중심$1~3
스탠다드실무 빈번 사용자 (부서장 추천)800회Haiku + Sonnet$4~8
프로기획·분석 헤비유저 (승인제)2,000회Sonnet 중심$15~35

* 2026년 6월 기준 Claude API 단가: Haiku 4.5 $1/$5, Sonnet 4.6 $3/$15 (입력/출력, 백만 토큰당). 단가 변동 시 재산정.

  • 운영 규칙 — 미사용 크레딧 이월 없음 / 소진 시 부서장 승인으로 추가 부여 / 매월 1일 자동 리셋
  • 등급 조정 — 분기별 사용 실적 리뷰로 상향·하향 (Usage API 리포트 기반)
  • 공용 작업 — 정기 리포트 자동화 등 시스템 작업은 개인 크레딧과 분리된 별도 Workspace로 운영

5비용 시뮬레이션 (30~100명)

가정: 활성 사용자 1인 1일 평균 15회 질의, 월 21일 근무, 회당 입력 2,000·출력 800토큰, 모델 비중 Haiku 70% : Sonnet 30%, 전 직원의 약 70%가 활성 사용.

예산 안전장치 Anthropic 콘솔에서 Workspace별 월 지출한도(Spend Limit)와 분당 요청·토큰 한도를 설정할 수 있어, 어떤 경우에도 설정 예산을 초과 과금되지 않습니다. 또한 Usage & Cost API로 모델별·Workspace별 사용량을 자동 집계하여 월간 리포트를 생성합니다.
