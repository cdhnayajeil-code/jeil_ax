# JEIL 전사 AX 비용 기획 보고서
### API 토큰 방식 vs 팀빌딩(좌석제) 방식 비교 · 비용 산정 · 헤비유저 분리 과금 설계

> 작성 기준일: 2026-06-18 · 대상: 전사 100~300명 · 주력 Claude(Anthropic), 보조 GPT(OpenAI API 연동)
> 환율 가정: **₩1,400 / USD** (예산 보수적 적용) · 모든 단가는 100만 토큰당 USD

---

## 1. 요약 (Executive Summary)

전사 AX를 위한 AI 사용 방식은 크게 **(A) 토큰형 API 방식**과 **(B) 팀빌딩(좌석제) 방식** 두 가지다. 두 방식은 과금 구조가 완전히 다르다.

- **좌석제**는 사람 수 × 정액으로 예측이 쉽지만, 안 쓰는 인원도 비용이 나가고 사용량 한도가 정해져 있다.
- **API 방식**은 실제 쓴 토큰만큼만 내므로 저사용 인원이 많은 전사 환경에서 총비용이 낮고, **부서별·개인별 사용량 추적과 별도 과금이 가능**하다.

JEIL은 이미 사내 포털(ai.jeilm.co.kr)을 API로 구축 중이므로, **API 게이트웨이 중심 + 헤비유저 분리 과금** 구조가 가장 적합하다. 200명 기준 월 비용은 모델 라우팅 적용 시 **약 ₩296만(연 약 3,550만 원)** 수준으로, 동일 인원 전원 좌석제(₩560만/월) 대비 **약 47% 절감**된다.

핵심 결론 세 가지:
1. **주력은 API 방식**으로 가되, 헤비/전문 사용자만 선택적으로 좌석(Premium) 또는 별도 쿼터를 부여하는 **하이브리드**가 비용 효율이 가장 높다.
2. **헤비유저 별도 과금은 기술적으로 가능**하다. Anthropic Admin(Usage & Cost) API와 Workspace/API Key 단위 한도, OpenAI의 Project 단위 한도를 쓰면 부서·개인별 비용 귀속과 월 상한 설정이 된다.
3. **모델 라우팅**(간단 작업은 Haiku/mini, 일반은 Sonnet, 고난도만 Opus/GPT-5.5)이 단일 모델 대비 비용을 25~40% 추가 절감한다.

## 2. 두 가지 방식 비교

| 구분 | (A) 토큰형 API 방식 | (B) 팀빌딩(좌석제) 방식 |
|---|---|---|
| 과금 기준 | 실제 사용한 입력/출력 **토큰량** | **사용자 1인당 정액**(월/연) |
| 비용 예측성 | 사용량에 비례(변동) | 매우 높음(고정) |
| 저사용자 비용 | 거의 0 (쓴 만큼만) | 안 써도 좌석 비용 발생 |
| 사용량 한도 | 한도 직접 설정 가능 | 플랜별 한도 내 사용 |
| 사내 시스템 통합 | 자유(포털·ERP·자동화에 직접 호출) | 제한적(웹/앱 중심) |
| 개인·부서별 비용 추적 | **가능**(Key/Workspace 단위) | 좌석 수 기준만 |
| 헤비유저 분리 과금 | **가능**(한도/별도 Key) | Premium 좌석으로 일부 가능 |
| 적합 대상 | 사내 앱·자동화·전사 변동 사용 | 고정적으로 매일 쓰는 전문 인력 |

**해석:** 전사 100~300명 중 상당수가 "가끔 쓰는" 사용자인 점을 감안하면, 전원 좌석제는 과투자가 된다. 사내 포털을 API로 운영하면 저사용자 비용이 사실상 0에 수렴해 총비용이 낮아진다.

## 7. 헤비유저 별도 과금 — 가능 여부와 방법

**결론: 가능하다.** 전사가 하나의 풀(pool)을 공유하더라도, 사용량을 개인·부서 단위로 추적하고 한도를 걸 수 있다.

### 7-1. Claude(Anthropic)에서 제공하는 통제 수단
- **Usage & Cost Admin API**: 조직 전체 토큰 사용량을 **모델별·Workspace별·API Key별**로 조회. 입력/캐시/출력 토큰을 구분해 부서·개인별 비용 귀속이 가능.
- **Workspace(Project) 단위 분리**: 부서별 Workspace를 만들고 각 Workspace에 **월 지출 한도(spend limit)** 설정.
- **Rate limit(분당 요청/토큰, 일일 토큰 상한)**: 특정 사용자·키의 과다 사용 방지.

### 7-2. OpenAI(GPT)에서 제공하는 통제 수단
- **Project 단위 분리 + 사용량/지출 한도**: 부서·용도별 Project로 키를 나누고 상한 설정.
- **API Key별 사용량 집계**로 비용 귀속.

### 7-3. JEIL 적용 설계 (사내 포털 기준)
사내 포털이 API를 중계하므로, **사용자 인증(Entra SSO)과 토큰 사용량을 포털 단에서 직접 로깅**하는 것이 가장 정밀하다.

1. 모든 요청에 사번/부서 태그를 붙여 사용량 DB(ERP 측 MSSQL 또는 별도 테이블)에 기록.
2. **월 기본 쿼터**(예: 1인당 200만 토큰/월)를 부여하고, 초과분은 별도 정책 적용:
   - (옵션 A) **부서 예산에 차지(chargeback)** — 초과 토큰 비용을 부서 비용센터로 배분.
   - (옵션 B) **승인제 한도 상향** — 초과 시 관리자 승인 후 한도 증액.
   - (옵션 C) **헤비유저 좌석 전환** — 상시 헤비 사용자는 Claude Team **Premium 좌석**으로 이전(정액화).
3. 대시보드에서 부서별·개인별 Top 사용자, 월 비용 추이를 시각화(이미 구축된 니즈조사 대시보드 인프라 재활용 가능).

### 7-4. 분리 과금 구축에 드는 추가 비용
- **Anthropic/OpenAI 측 추가 요금: 없음.** Usage/Cost API와 Project 한도는 기본 제공 기능이다.
- **사내 구축 비용(1회성)**: 사용량 로깅·쿼터·대시보드 개발 공수. 기존 포털/대시보드에 기능 추가 수준이면 **수일~2주 내** 개발 가능(별도 라이선스 비용 없음).
- **운영 비용**: 사용량 DB 저장/조회 비용(미미). 헤비유저를 Premium 좌석으로 돌릴 경우 좌석당 연납 $100/월(약 ₩14만) 추가.

## 8. 권고 기획안

**하이브리드 + API 게이트웨이 중심 모델**

1. **기본은 API 방식**: 사내 포털(ai.jeilm.co.kr)이 Claude/GPT를 중계. 전 직원 접근 허용, 저사용자는 사실상 무비용.
2. **모델 라우팅 도입**: 단순 작업 Haiku/GPT-5.5 mini → 일반 Sonnet → 고난도만 Opus/GPT-5.5. 프롬프트 캐싱·출력 길이 가이드 병행.
3. **사용량 거버넌스**: 사번·부서 태깅 로깅 + 1인 월 기본 쿼터 + 초과 시 부서 차지백/승인제.
4. **헤비유저 선별 좌석화**: 매일 대량 사용하는 전문 인력(기획·개발·분석 등 소수)만 Team Premium 좌석으로 정액화해 비용 예측성 확보.
5. **월 예산 가드레일**: Workspace/Project 지출 한도로 전사 월 상한을 하드캡 설정(예상 초과 방지).

**예상 200명 총비용:** 월 약 ₩296만(API 라우팅) + 헤비 좌석 일부 ≈ **월 ₩300만~400만, 연 ₩3,600만~4,800만** 수준에서 운영 가능.

## 9. 가정 및 주의사항

- 단가는 2026-06 공개 기준이며 공급사 정책에 따라 변동될 수 있음. 계약 전 콘솔/영업 최종 확인 필요.
- 환율 ₩1,400/USD 가정. 실제 청구는 USD 기준이라 환율 변동 시 KRW 비용이 달라짐.
- 사용 프로파일(일 질의 수·토큰량)은 추정치다. 실제 1~2개월 파일럿 로그로 보정 권장.
- Claude **Enterprise** 플랜은 비공개 협상가(시트당 약 $60, 일부 계약 최소 70석 보고)로, 대규모 전환 시 별도 견적 필요.
- 좌석제는 좌석 정액에 일정 사용량이 포함되나 한도 초과분은 API 단가로 추가 과금될 수 있음(플랜별 상이).
